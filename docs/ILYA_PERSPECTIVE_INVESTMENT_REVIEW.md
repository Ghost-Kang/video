# Cascade / OpenRHTV · Ilya 视角投入评估

**Date**: 2026-05-19  
**Scope**: 基于 `docs/` 项目文档，对 Cascade / OpenRHTV 的价值、前景、竞争力与资源投入方式做判断。  
**结论**: 值得验证，不值得直接全量开发。当前计划太多 scale，不够 research。

---

## 1. 核心判断

这个项目有真实价值。

但当前计划太大。太多 scale，不够 research。

Cascade 不应该被理解为“AI 视频生成器”。如果只是这个，它很弱。Seedance、Kling、Runway、即梦、CapCut 会持续把视频生成、角色一致性、多镜头、音频、编辑能力做成基础设施。

Cascade 真正可能有价值的定义是：

> **把中文短视频文化压缩成可复用的创作结构。**

爆款视频不只是画面。

它有：

- hook
- 节奏
- 情绪曲线
- 社会语境
- 角色模式
- 传播原因

如果 Cascade 能把一条抖音 / 小红书爆款压缩成“某个创作者赛道可复用的公式”，那它在做有价值的事。

---

## 2. 价值在哪里

最值得投入的是：

1. 理解为什么一条内容火。
2. 改写到用户自己的赛道。
3. 沉淀可复用角色 / 场景锚点。
4. 从多次创作中学习用户的内容分布。

这不是简单生成。

这是压缩。

```text
原始爆款内容
  -> 结构化理解
  -> 赛道适配
  -> 用户资产
  -> 下一次更好的生成
```

如果这个循环成立，Cascade 有复利。

如果每次创作都是独立 prompt chain，Cascade 就没有学习。

没有学习，就只是昂贵的 wrapper。

---

## 3. 不值得投入什么

最不值得投入的是：

- 和大模型平台比视频生成质量
- 一开始做完整 60s 自动成片
- 一开始做完整后台
- 一开始做创作者数据中心
- 一开始做 B2B 报告
- 一开始做复杂商业化
- 一开始做多模式画布和复杂 Agent 触发器

这些不是永远不做。

只是现在不做。

当前真正要证明的不是“能不能生成视频”。

真正要证明的是：

> 能不能把爆款结构压缩成用户自己的创作程序。

---

## 4. 竞争力判断

Cascade 不能在模型层竞争。

模型层会被更大的实验室和平台持续推进：

- Seedance / 即梦
- Kling / 可灵
- Runway
- Sora
- CapCut / 剪映

这些系统会继续提高：

- 视频质量
- 角色一致性
- 场景一致性
- 多镜头控制
- 原生音频
- 编辑能力
- 参考图 / 参考视频能力

Cascade 的竞争力只能在更高层：

1. 中文短视频语义。
2. 爆款结构抽取。
3. 赛道特定改写。
4. 用户创作记忆。
5. 锚点资产复用。
6. 真实创作反馈循环。

这不是“生成能力”的竞争。

这是“学习能力”的竞争。

---

## 5. 最大担忧

当前 MVP 范围太大。

`MVP_SCOPE.md` 估算到 v2 约 160 人天，包含：

- 视频分析
- 多数据源
- hotspot
- toprador
- Newrank
- LangGraph
- 画布
- 锚点
- TTS
- BGM
- 字幕
- ffmpeg
- 数据后台
- 创作者数据中心
- 商业化 hook

这不是一个产品假设。

这是很多假设叠在一起。

更重要的是，最关键的上游表示还不稳定。

`TOPRADOR_SCHEMA_INVESTIGATION.md` 已经说明，`analysis_result` 的实际 schema 还没有完全枚举：

- `viral_analysis` 的 8 维字段未知
- `scenes[]` 字段未知
- 时间戳未知
- 角色字段未知
- 字幕字段未知
- 下载链路和失败模式未知

如果这个表示不稳定，下游的画布、字幕、prompt、锚点都会脆弱。

先稳定表示。

再做产品。

---

## 6. 建议的 4-6 周验证

我会先做一个研究验证，而不是完整 MVP。

目标：

> 能否可靠地把 20 条真实短视频转化为有用的创作结构？

### 6.1 验证步骤

1. 取 20 条真实抖音 / 小红书 URL。
2. 结构化分析爆款原因。
3. 测 `analysis_result` 字段完整率。
4. 生成用户赛道改写版本。
5. 产出 3-5 个镜头草稿。
6. 找 10 个真实创作者试用。
7. 统计失败类型和成本。

### 6.2 暂时不做

- 完整 60s 自动成片
- 完整 `/admin`
- `/me/dashboard`
- B2B 报告
- 多模式画布
- 完整 TTS / BGM / 字幕链路
- marketplace
- 复杂付费系统

---

## 7. 继续投入 Gate

继续投入的 gate：

| 指标 | 通过线 |
|---|---:|
| 深分析成功率 | >= 80% |
| 核心字段完整率 | >= 90% |
| 真实创作者测试 | 10 人 |
| 完成第一条内容 | >= 5 人 |
| 一周内回来做第二条 | >= 3 人 |
| 用户明确说“它帮我决定拍什么” | >= 2 人 |
| 成本 | 能支撑 ¥39/月 |

如果这些通过，再进入完整 MVP。

如果不通过，不要继续堆功能。

需要回到核心问题：

- 爆款结构是否能被稳定提取？
- 赛道改写是否真的有用？
- 用户是否理解并使用锚点资产？
- 系统是否从用户重复创作中学习？

---

## 8. 最终结论

Cascade 值得验证，不值得直接全量开发。

它的机会不是 AI 视频生成。

它的机会是：

```text
把中文短视频爆款
  压缩成结构
  改写成赛道程序
  沉淀为用户资产
  并在重复创作中学习
```

如果这件事成立，Cascade 值得继续。

如果不能，画布、锚点、Agent、数据看板都只是表面积。

想法比资源重要。

这里值得保留的想法是：

> **learning-and-adaptation loop**

先证明这个循环。

再扩展其他东西。
