# Cascade / OpenRHTV · Naval 视角投入评估

**Date**: 2026-05-19  
**Scope**: 基于 `docs/` 项目文档，对 Cascade / OpenRHTV 的价值、前景、竞争力与资源投入方式做判断。  
**结论**: 值得验证，不值得全量开发。先找到 specific knowledge，再加杠杆。

---

## 1. 先重新定义这个项目

如果 Cascade 是“AI 视频生成器”，它不值得重投入。

模型平台会吃掉它。Runway、Kling、Seedance、Sora、即梦、CapCut 这类平台会持续把画质、角色一致性、多镜头、配音、剪辑等能力做成基础设施。

你不应该在模型层竞争。那是资本杠杆游戏，不是这个项目当前最强的 specific knowledge。

但如果 Cascade 是：

> **中文短视频创作者的选题到成片操作系统**

它有价值。

它真正的链路不是“输入 prompt -> 生成视频”，而是：

```text
热点发现
  -> 爆款理解
  -> 赛道改写
  -> 锚点复用
  -> 发布包
```

这个链路替用户做了一个更底层的判断：

> 今天什么值得拍，以及它如何变成我的版本。

---

## 2. 价值判断

Cascade 有价值，因为它有机会结合两种 permissionless leverage：

1. **代码杠杆**：自动化热点分析、脚本生成、视频草稿、锚点复用。
2. **媒体杠杆**：帮助创作者持续产出可传播内容。

每次使用如果只生成一条视频，那价值很浅。

每次使用如果能沉淀资产，就有复利：

- **赛道索引**：用户自己的内容判断系统。
- **锚点资产**：角色、场景、风格可复用。
- **创作数据**：哪些热点、模板、角色、表达方式真的转化。

工具会被替代。资产会复利。

Cascade 要做的是资产，不是一次性工具。

---

## 3. 前景判断

前景中等偏高，但前提是缩小楔子。

中文创作者的痛点不是“我不会生成视频”，而是：

```text
我不知道拍什么。
我不知道为什么别人会火。
我不知道怎么改成自己的赛道。
我拍完以后没有复利资产。
```

如果 Cascade 能证明它让创作者更快找到可拍选题，并且一周内重复使用，它就有前景。

如果它只是把 Runway / Kling / Seedance / 即梦包装成一个画布，它没有前景。

那不是复利。那是接口聚合。

---

## 4. 竞争力判断

### 4.1 不要在模型质量上竞争

模型质量不是 Cascade 的护城河。

大模型视频平台会持续提高：

- 角色一致性
- 场景一致性
- 多镜头叙事
- 原生音频
- 编辑控制
- 参考图 / 参考视频能力

这些能力会变成基础设施。基础设施层需要资本、算力、人才和分发。Cascade 不应该在这里打。

### 4.2 真正的竞争力

Cascade 的竞争力只能来自更靠近用户语境的地方：

1. 更懂中文短视频生态。
2. 更懂抖音 / 小红书爆款复刻。
3. 能把热点数据翻译成创作动作。
4. 能让创作者沉淀自己的赛道知识。
5. 能让角色和场景跨视频复用。

如果这五点成立，Cascade 不是在和生成器竞争。

它是在和旧流程竞争：

```text
刷抖音
  -> 截图
  -> 自己脑补
  -> 写脚本
  -> 切剪映
  -> 反复试错
```

旧流程很低效。这里有机会。

### 4.3 对不同竞品的判断

| 竞品类型 | 强项 | Cascade 的正确位置 |
|---|---|---|
| Runway / Kling / Seedance / Sora / 即梦 | 模型质量、生成能力、视频效果 | 当 provider，不正面竞争 |
| CapCut / 剪映 | 编辑心智、用户基础、发布链路 | 避开编辑后端，切选题和爆款学习前端 |
| 新榜 / 飞瓜 / 卡思 | 数据和榜单 | 把数据变成创作动作 |
| MCN 内部流程 | 业务语境、团队执行 | 提供标准化、可复用、可报告的工作流 |

## 5. 最大风险：把自己做成一份更复杂的工作

当前 MVP 范围太大。

`MVP_SCOPE.md` 估算到 v2 已经是约 160 人天，约 8 个月。范围包括：

- 视频分析
- 多 provider 视频生成
- 锚点系统
- 画布系统
- TTS
- BGM
- 字幕
- ffmpeg 合成
- 热点数据
- hotspot 算法中台
- 新榜 sidecar
- toprador sidecar
- Agent 触发器
- 数据后台
- 创作者数据中心
- B2B 报告
- 商业化与配额

这会把项目变成一个低杠杆的服务公司。

你会不停修接口、修 UI、修数据、修 provider、修运营后台。

这不是 wealth。  
这是 employment with extra steps.

## 6. 值得投入什么

值得投入一个 6 周验证版。

不是 6 个月平台版。

### 6.1 6 周只验证一个问题

> 用户会不会因为 Cascade 更快找到值得拍的题，并真的重复创作？

### 6.2 最小版本

只做这条闭环：

```text
输入热点 URL 或话题
  -> 分析为什么火
  -> 改成用户赛道版本
  -> 生成脚本
  -> 生成 3-5 个镜头草稿
  -> 复用一个角色或场景
  -> 导出标题、标签、脚本、素材路径
```

### 6.3 先砍掉

- 完整 `/admin`
- `/me/dashboard`
- 6 Tab `/topics`
- 精修模式
- 手动模式
- B2B 报告
- marketplace
- 复杂 Agent 5 触发器
- 完整商业化系统
- 完整多租户
- 完整数据中台

这些不是不重要。

它们只是现在不重要。

---

## 7. 继续投入 Gate

6 周后，只看行为，不看愿景。

| 指标 | 通过线 |
|---|---:|
| 真实创作者试用 | 10 人 |
| 完成第一条内容 | >= 5 人 |
| 一周内做第二条 | >= 3 人 |
| 单条成本 | < ¥15 |
| 热点进入创作转化率 | >= 15% |
| 用户明确说“比直接用即梦 / 可灵更有用” | >= 2 人 |

达不到，就停。

不是继续做更多功能。

要回到核心问题：

- 用户真的从热点开始吗？
- 爆款改写是否有价值？
- 赛道索引是不是用户能理解的东西？
- 锚点复用有没有被感知？
- 生成质量是否足够支撑发布？
- 成本是否允许 ¥39/月模型成立？

## 8. 资源投入建议

### 当前阶段

建议投入：

- 1 名全栈工程
- 创始人自己做产品判断
- 设计只做轻量可用，不做完整设计系统
- 6 周
- 10 位真实创作者

不建议现在投入：

- B2B 销售
- 完整设计团队
- 完整 BI
- 支付深度建设
- 多人协作
- marketplace
- 大规模数据中台

### 下一阶段

只有通过 Gate 后，再进入 3 个月产品化：

1. 稳定视频生成链路。
2. 做标准模式画布。
3. 强化锚点复用。
4. 完善 topics 雷达。
5. 加基础付费和配额。
6. 招募 30 人 Beta。

---

## 9. 最终结论

Cascade 值得验证，不值得全量开发。

如果它是生成器，会输。

如果它是中文创作者的：

```text
选题判断系统
  + 爆款复刻系统
  + 赛道资产层
  + 可复用创作资产库
```

它有机会。

先找到 specific knowledge：

> 你是否真的比模型平台更懂中文创作者如何从热点变成自己的内容？

如果答案是 yes，再加代码和媒体杠杆。

不要反过来。
