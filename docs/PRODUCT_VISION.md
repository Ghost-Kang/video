## 0. 一句话定位

> **Cascade 是面向中文个人创作者的 AI 视频创作平台。它把"看懂别人的爆款"和"做自己的版本"合二为一——从热点雷达到画布创作一条龙，用"锚点为主角"的画布让用户感觉自己在导演一个虚拟剧组。**

---

## 1. Why Now / 为什么是现在

### 1.1 市场现状

| 产品 | 强项 | 缺什么 |
|---|---|---|
| RHTV | 画布 + agent + 锚点的"虚拟剧组"思路 | 已无后续运营；产品形态学习成本高 |
| 即梦 / 可灵 / Sora 2 | 单条生成质量高、character reference 能力 | **只有生成，没有"发现 + 学习"链路** |
| Heygen / Synthesia | Talking head 工业化 | 单一形态、不适合中文内容生态 |
| Pika / Runway | 视觉特效 | 离中文短视频生态远 |
| 剪映 / CapCut | 后期剪辑 | 不是从零生成 |
| **新榜 / 飞瓜 / 卡思** | 热度数据 | **只看数据、不能创作** |

**没人在做的事**：把**热点发现** + **爆款学习** + **个性化复刻** 三件事用 agent 驱动的画布串起来。

### 1.2 我们的独特性

**别人有的我们也有**（不是壁垒，是入场券）：
- 视频帧生成（Apimart / Google Gemini）
- 视频片段生成（Kling / Seedance）
- 字幕、TTS、BGM、剪辑合成

**别人没有、我们 day-1 就有**（这才是护城河）：

1. **锚点为主角的画布**——character / scene 锚点不是隐藏字段，是画布上明确可视的"虚拟剧组成员"。用户看到自己在"管理一个剧组"，不是在"填表格"
2. **从抖音 URL 学习 → 复刻**——粘贴任意爆款链接，agent 拆分镜级、拿着 viral_analysis 帮你做"风格化复刻"
3. **三数据源融合的热点雷达**——60s 实时（话题面）+ 新榜深度（视频元数据）+ toprador 深分析（视频内容）一站打通
4. **Agent 智能弹起**——5 个关键决策点主动出现，不打扰、不消失

### 1.3 产品定位

**与现有 ACMM 关系**：平行新立项。ACMM 保留为 B2B 营销团队 SaaS；Cascade 面向个人 + MCN + 品牌甲方。

**主战场**（按优先级）：
1. **个人创作者 / 自媒体**（MVP 增长锁定） — 推特 / 小红书 KOL 圈、创始人个人品牌驱动获客
2. **MCN / 内容工作室**（V1.5 变现起势） — 项目层 + 资产复用 + 团队协作
3. **品牌甲方**（V2+ 高客单） — 品牌库 + 合规 + 审稿流

## 2. 目标用户画像

### 2.1 个人创作者（MVP 锁定）

**典型画像**："30 岁宝妈，做辅食 + 育儿日常，主要在小红书和抖音"
- **痛点**：内容枯竭、跟不上热点、单条视频做 4 小时
- **核心诉求**：3 个月内做出一条爆款（即"被使用 + 重复使用"）
- **付费意愿**：¥39/月可接受，¥99/月以上需要明确价值
- **学习意愿**：会看 1-2 分钟教程，不会读说明书
- **设备**：电脑 + 手机各 50%（MVP 锁定电脑端，手机端 V2）

### 2.2 MCN / 内容工作室（V1.5）

**典型画像**：3-10 人小团队，运营 5+ 个 IP 矩阵，月产 100+ 条
- **痛点**：IP 角色形象漂移、新人上手慢、客户报告繁琐
- **核心诉求**：角色一致性 + 模板复用 + 批量
- **付费意愿**：¥500-2000/月座位制
- **关键功能需求**：项目层、资产库、模板库、团队协作

### 2.3 品牌甲方（V2+）

**典型画像**：消费品 / 美妆 / 3C 品牌内容部门
- **痛点**：代理商沟通成本、风格不可控、版权合规
- **核心诉求**：品牌一致性 + 合规审查 + 版本管理
- **付费意愿**：年单 5-50 万
- **关键功能需求**：品牌库（字体/Logo/调色板/禁用词）、审稿流、版本对比

### 3.2 创作旅程：从一句话到一支视频

```
USER: 一句话 / 上传参考图 / 选热点话题
   │
   ▼
AGENT 智能弹起（① 初始问风格）  ← 第 1 个智能弹起
   "你这条要走哪种调性？写实 / 动漫 / 赛博？"
   │
   ▼
进入画布
   │
   ├─ STAGE 1 · 剧本闸门（② agent 弹起）
   │   剧本生成 → 用户审核 → confirmed
   │
   ├─ STAGE 2 · 锚点闸门（③ agent 弹起）
   │   character 锚点（角色形象图） × N
   │   scene 锚点（场景参考图） × M
   │   用户对每个锚点审核 → confirmed
   │
   ├─ STAGE 3 · 镜头制作（4 个并行流）
   │   ├─ image-grounded 视频生成（Seedance + 锚点参考）
   │   ├─ TTS 配音（火山豆包）
   │   ├─ BGM 选取（关键词 → 素材库）
   │   └─ 字幕渲染（从 script.dialogue_and_narration）
   │
   ├─ STAGE 4 · 成片闸门（④ agent 弹起）
   │   ffmpeg 合成 MP4
   │   用户预览 → confirmed
   │
   └─ STAGE 5 · 发布（⑤ agent 弹起：导出/发布?）
       "成片完成。要复制发布包到剪贴板去抖音发，
        还是先导出 MP4？"
       │
       ▼
   一键复制发布包（MP4 + 标题 + 标签 + 封面） → 抖音/小红书 App
```

**Agent 触发点**（之前 12 题决策的固化）：
1. **初始问风格**（bottom sheet）
2. **剧本闸门**（modal）
3. **锚点闸门**（modal）
4. **成片闸门**（modal）
5. **出错时**（inline 卡片）
6. **完成问"导出/发布"**（bottom sheet）

详细 UI 形态见 `CANVAS_DESIGN.md` §5。

### 3.3 失败处置：4 选 1

视频生成是最高频失败场景。Agent 主动弹起 inline 卡片，给用户 4 个选项：

| 选项 | 后续动作 | 成本 | 适用 |
|---|---|---|---|
| 换个 prompt 重生 | agent 润色 prompt，用户看诗意变化 → 重生 | 同重生 | 用户不懂 prompt |
| 换个风格重生 | 切动漫/写实/赛博/油画 → 重生 | 同重生 | "词没错，风格不适" |
| 跳过这一镜 | 下游接受缺镜，成片少一个场景 | 0 | 不阻塞整体进度 |
| 完整重做从选题 | 本次 run 作废，从头来 | 整 run 重做 | 方向根本不对 |

详见 `CANVAS_DESIGN.md` §6。

### 3.4 数据看板三层（v2 新增 · 详见 `DATA_DASHBOARD.md`）

Cascade 三层数据统计模块，共享同一埋点底座：

| 层 | 读者 | 路径 | 形态 | MVP |
|---|---|---|---|---|
| **A · 运营管理后台** | 创始人 / 运营 / 财务 | `/admin/*` + Metabase | 混合（Metabase 深度查询 + /admin 8 核心 KPI 卡片） | ✅ M5 |
| **B · 创作者数据中心** | 单个创作者 | `/me/dashboard` | 独立页，4 模块（概览 / 趋势 / 成本拆解 / 用过热点+锚点复用） | ✅ M5 |
| **C · B2B 运营报告** | MCN / 品牌客户 | hotspot HTML 报告链接 | 已通过 hotspot 整合（V1.5） | V1.5 |

**埋点底座 day-1**：25 个核心事件 + `events` / `daily_metrics` / `user_funnels` 三表。**这是 6 个月后能不能看清产品赢在哪、输在哪的关键**。

### 3.5 三档模式：同一画布的三视图

> 数据模型统一，UI 渲染不同。**任何时候可切换**，"动中也能切"。

| 模式 | 入口 | 适用 | 画布表现 |
|---|---|---|---|
| ⚡ **闪电** | 入口页 ⚡按钮 / 输入框 GO | 5 分钟批量出片 | 进度条 + 缩略图流，agent 仅失败/完成时出现 |
| 🎯 **标准** | 入口页 🎯按钮（默认） | 15-30 分钟精创 | 横向画布，5 节点可视，三闸门主动弹起 |
| ✨ **精修** | 入口页 ✨按钮 | 1-2 小时精修 | 完整 React Flow，所有节点可点开深挖 |
| 🛠 **手动** | 入口页 🛠 按钮 | 从零自由创作（Figma 风） | 空画布，用户拖节点，agent 默认隐身可呼叫 |

**模式不绑定 run**，用户可随时切换：闪电跑到一半发现要精修，切过去看到所有 agent 自动决策（透明度 = 信任）。

## 4. 差异化锁点（深度解释）

### 4.1 "锚点为主角"是产品差异化，不是模型层差异化

| 同行 | 锚点做法 |
|---|---|
| 即梦 | character reference 是一个上传字段，藏在表单里 |
| 可灵 | 同上，character image 隐藏选项 |
| Sora 2 | character lock 隐式管理 |
| RHTV | 早期有锚点概念但产品没运营 |
| **Cascade** | **锚点是画布上明确的"虚拟剧组成员"，有自己的肖像、名字、状态** |

具体设计：
- 用户进画布看到 **"角色与场景库"** 侧栏 / 顶部 Tab，里面是已有的 character 锚点（小张、小李 ...）+ scene 锚点（公园早晨、办公室夜晚 ...）
- 创建新镜头时，可以从锚点库**拖**一个角色 + 一个场景到画布
- 每个锚点有自己的 "三视图"（front / side / back）、"风格定稿板"
- 锚点跨 run 复用——做完辅食视频里的"小张妈妈"，下次育儿视频可以直接用

### 4.2 "从抖音 URL 学习 → 复刻"是别人不敢做的杀手锏

- toprador 已实现：粘贴抖音链接 → 多模态 LLM 拆分镜 → 8 维 viral_analysis JSON
- 这能力 **整个 AI 视频创作赛道没人公开做** —— 因为大厂都怕版权
- Cascade 不是 "复制别人的视频"，而是 "学习别人的爆款公式 + 用你的锚点重做"
  - 用户上传抖音 URL → 分析 → "这条火是因为 hook 是 X、节奏是 Y、爆点是 Z"
  - agent 把 viral_analysis 灌进画布 → 用你的 character / scene 重做一条
  - 法律上是"借鉴爆款公式"，产品上是"个性化复刻"

详细数据流见 `TOPIC_TO_CREATION_PIPELINE.md`。

### 4.3 "四数据源融合 + hotspot 算法中台"是热点能力的差异化

| 数据源 | 数据特征 | Cascade 用法 |
|---|---|---|
| 60s 热搜（8+ 平台） | 实时话题词条 | "今天什么在火" — 首页轮播 + 灵感库 |
| 新榜 API（4 平台） | 视频级元数据（T-3） | "这话题下哪条具体视频火" — /topics 雷达深度卡片 |
| **hotspot 持久化时序数据**（4 张表） | **结构化时序**（hot_search_snapshot / rank_data / ai_video_info / ai_video_metrics） | **趋势对比 + 创作者画像 + 飙升话题检测** — 选题情报中台 |
| toprador video-analysis | 单条视频深度（分镜级） | "为什么这条火，怎么复刻" — on-demand 学习 |

### 4.4 hotspot 算法中台 — 业界唯一工业级选题输出

不止有数据，还有**工业级算法**（OPPO 内部已生产验证）：

| 算法 | 输出 | 同行有吗 |
|---|---|---|
| **双轨评分**（泛热点 / 品牌契合） | 0-100 分数 + 维度分解（热度/排名/可玩法/传播 OR 热度/功能匹配/场景落地/品牌契合） | 没有 |
| **五维评分**（话题热度 / 趋势加速度 / 内容效率 / 品牌契合 / 竞争空位） | 综合得分 + 蓝海/红海地图 | 没有 |
| **互动 DNA 四格** | 点赞/收藏/评论/分享主导消费方式 | 没有 |
| **可复刻性 0-100** | 直接告诉创作者"这条好不好做" | 没有 |
| **最佳时长推荐** | 6 档桶聚合（≤15s / 15-30s / ...）+ 数据驱动 | 没有 |
| **飙升话题检测** | 同环比增长率 | 没有 |
| **竞争空位地图** | 蓝海/机会型/常规/红海 | 没有 |
| **创作者雷达** | 常驻爆款 / 素人黑马 / 赛道专精 | 没有 |
| **三池过滤**（轻松泛大众优先 / 硬核降权 / 八卦过滤） | 选题质量保障 | 没有 |
| **主题驱动三轨流水线** | 用户给主题词 → 多源自动过滤打分 | 没有 |

**对比 ACMM 现有 topic-analysis（3 句话浅分析）**：hotspot 是**客观确定性算法**输出可量化指标，ACMM 是**主观 LLM**输出叙事文本。两者**并存**——hotspot 给底层"分数 + DNA + 时长 + 可复刻"，ACMM AI 在顶部给"为什么火 + 怎么改"人话翻译。

详见 `TOPIC_TO_CREATION_PIPELINE.md` §2-§5。

## 5. 商业化路线展望

> MVP 不拍定价，但 **schema 层 day-1 预留所有商业化 hook**。

### 5.1 创作侧 revenue streams

| Stream | 定位 | MVP | 关键字段 |
|---|---|---|---|
| **基础订阅（freemium → Pro → Team → Enterprise）** | 主收入 | ✅ ¥39/月起 30 条 | `planTier`, `quotaUsed`, `quotaTotal` |
| **按使用量加包** | 超额加购 | 🟡 schema 备 | `costFen`, `usageMeterId` |
| **TTS / BGM 高级 provider** | 解锁高级模型 | ⏸ V2 | `provider`, `voiceTier` |
| **模板商城** | 平台抽成（运营 + UGC） | 🟡 MVP 仅运营 | `templateOrigin`, `parentTemplate`, `revShareUserId` |
| **资产市场** | 角色 / 场景 / BGM 付费内容 | ⏸ V2 | `assetType`, `licenseTier` |
| **创作者作品被复刻分成** | UGC 经济飞轮 | ⏸ V2 | `parentRunId`, `revShareTier` |

### 5.2 热点选题侧 revenue streams（**v2 新增重点**）

这是与 ACMM 没有的全新变现 surface：

| Stream | 定位 | MVP | 商业逻辑 |
|---|---|---|---|
| **数据源订阅分层** | freemium 仅 60s（开源免费），Pro 解锁新榜（每 key ¥/月），Enterprise 自定义数据源 | ✅ 三层架构 day-1 备好 | 新榜 API key 实际有成本，需要 freemium 限次 |
| **AI 分析按次** | 浅分析（whyItHit）免费、深分析（toprador full analysis）按次 / Pro 配额 | ✅ 渐进式 UX 天然分层 | 多模态 LLM 单次 ~¥0.3-0.8，需要 metering |
| **赛道索引订阅**（hotspot 整合关键 · v3 新增） | 个人 = 1 赛道索引免费、Pro = 5 赛道、Team = N 赛道、Enterprise = 品牌库（含功能映射 + 场景落地 + 经验风格） | 🟡 schema day-1 备好 | 把 hotspot 的 ColorOS 功能索引模式产品化——MCN 上传"赛道词表 + 场景映射"，品牌方上传"功能索引 + 禁用词"。**hotspot 整合的最大商业化 hook** |

| **舆情报告** | 月度/周度自动生成行业舆情 PDF | ⏸ V3 | Enterprise 增值 |

## 6. 与现有项目的关系

### 6.1 OpenRHTV（前身 / 原型）

- **角色**：Cascade 的技术骨架来源
- **复用**：画布 + LangGraph agent + 锚点系统 + Image gen providers + WebSocket 协议 = **100% 沿用**
- **演进**：单租户全局变量 → 多租户 schema；SQLite → Postgres；canvas-only → 完整工坊

### 6.2 toprador / video-analysis（视频后端 base）

- **角色**：Cascade 视频生产能力的来源
- **复用**：视频分析、Kling、Seedance、剧本生成、字幕、视频合成、60s、Douyin_Download = **整套搬，脱敏**
- **演进**：企业 SSO → Clerk-style 公开鉴权；OPPO 内部 LLM 网关 → 公网 LLM（豆包/通义/智谱）；MySQL → Postgres
- **使用权确认**：✅ 已确认可完整使用（脱敏后开源化部署）

### 6.3 ACMM（独立 B2B 线 + 部分模块借鉴）

- **角色**：保持独立，B2B 营销团队线继续运营
- **借鉴**：热门选题雷达模块（newrank 接入 + topic LLM 分析） = **TS 整套移植，作为 Node sidecar 接入**
- **不合并**：两个产品平行运营、不同品牌、不同定价、不同用户群

### 6.4 hotspot（选题情报中台 base）

- **角色**：Cascade **选题情报中台**——所有选题数据 + 算法 + 报告生成
- **复用**：4 张数据表 + 9 个 Skills + 全部算法（双轨/五维/DNA/可复刻/飙升/竞争空位/创作者雷达/三池过滤/主题流水线/HTML 报告生成）= **整套搬，脱敏 + 去 ColorOS 化**
- **演进**：MySQL → Postgres；ColorOS 功能索引 → 用户自定义"赛道索引"架构；HTML 报告作 V1.5 B2B 增值产品
- **架构定位**：作为独立 Python sidecar 部署（与 toprador / 60s / newrank-sidecar 同级）
- **使用权确认**：✅ 已确认完全可用（脱敏后公网部署）
- **整合方案**：方案 A（整套作为 Cascade 选题情报中台）

## 8. 反共识假设（要主动验证的）

这些假设如果错了、整个产品形态会变：

1. **"画布 + agent" 对个人创作者不是负担**——闪电视图能否消化掉学习成本？
2. **"锚点为主角"用户感受得到差异化**——而不是变成又一个"上传参考图"字段
3. **"从抖音 URL 学习"会被市场接受**——版权/法律风险是产品级风险（不是个人风险）
4. **三档模式不会让首屏选择困难症爆发**——Hick's law 风险
5. **三数据源融合用户能消化**——选择困难 vs 一站式价值的拉锯
6. **"一句话直发" + 智能弹起 5 触发器 = 合适的自动化粒度**——而不是每一步都被打断的烦人

每个假设的验证路径见 `MVP_SCOPE.md` §6 实验设计章节。
