## 3. 真共识（6 位全部同意）

以下每一条都至少在 5/6 评审中显式出现，且无任何评审反对——视为不可违反的 Phase 1 边界：

1. **定位收敛**：Cascade 不是 AI 视频生成器，是「把别人的爆款变成你的版本」的中文创作者工作系统（Buffett §2 / Elon §1 / Naval §1 / Jobs §1 / Ilya §1；Karpathy 不反对）。
2. **不在模型层竞争**：把 Kling / Seedance / 即梦 / Runway / Sora 当 provider，竞争层在工作流和资产复用（Buffett §2 / Elon §4.1 / Naval §4.1 / Jobs §3 / Ilya §4；Karpathy 不反对）。
3. **砍掉 6-8 个月 / ~160 人天大计划**，改为 4-6 周单闭环验证版（全部 6 位）。
4. **Phase 1 唯一保留的闭环** = 粘贴 URL → 浅分析「为什么火」→ 选赛道 → 改写脚本 → 3-5 镜头草稿 → 复用 1 角色 + 1 场景 → 导出发布包（Buffett §7 / Elon §6 / Naval §6 / Jobs §7 / Ilya §6；Karpathy §5 Week 1-6 与此一致）。
5. **行为型 Gate**：≥ 10 真实创作者 / ≥ 5 完成首条 / ≥ 3 一周内回访（Buffett §8 / Elon §7 / Naval §7 / Jobs §9 / Ilya §7；Karpathy 不反对）。
6. **单位经济 Gate**：单条成本 < ¥15，支撑 ¥39/月订阅（全部 6 位）。
7. **明确「不做」清单**：完整 60s 自动成片 / `/admin` / `/me/dashboard` / 6 Tab `/topics` / 多模式画布 / 5 Agent 触发器 / B2B / marketplace / 多租户 / OAuth 一键发布（5/6 显式，Karpathy 不反对）。
8. **关键节点必须有人类兜底**，不做完全 URL → 全自动成片（Elon §6.3 / Naval §6 / Jobs §7 / Ilya §3 / Karpathy §3.8；Buffett 隐含同意）。
9. **Gate 不通过 → 停 + 回核心假设排查，不堆功能**（Buffett §8 / Elon §7 / Naval §7 / Jobs §9 / Ilya §7；Karpathy §4 一致）。

## 6. Top 5 Phase 1 不可妥协的红线

按「若违反 → 整个 6 周实验失去信号」排序：

### 🔴 1. 上游 `analysis_result` schema 必须先稳，再做任何下游

- **依据**：Karpathy §1 全文 + Buffett §5「基础数据不稳定 → 随机体验 → 无复购」+ Ilya §5「先稳定表示，再做产品」。
- **可检验门**：核心字段完整率 ≥ 90%、深分析成功率 ≥ 80%、静默失败 = 0，否则不进 Phase 1。
- **违反后果**：画布、字幕、prompt、锚点全部建立在 LLM 梦境上，Phase 1 的 Gate 全部失去意义。

### 🔴 2. 定位收敛为单句「把别人的爆款变成你的版本」，所有 Phase 1 功能都必须服务这一句话

- **依据**：Jobs §1 + Buffett §2 + Elon §1 + Naval §1 + Ilya §1（5/6 显式，1/6 不反对）。
- **可检验门**：Phase 1 任何一个 PR / feature 都能用这句话辩护，否则砍掉。Gate 中「≥ 2 人能一句话说出价值」是底线。
- **违反后果**：复杂度回归，Cascade 退化为「又一个 AI 视频生成器」，进入红海。

### 🔴 3. 6 周内只验证单一闭环（URL → 改写 → 草稿 → 复用 → 发布包），且每个关键节点保留人类兜底

- **依据**：全部 6 位评审。Elon「先造发动机不建火星城市」、Karpathy「自动化越强 schema 风险越大」、Buffett「不要先做完整数据中台」。
- **可检验门**：所有「不做」清单（`PHASED_PLAN §4.3`）必须遵守；任何「顺手再加一点」的诱惑都拒绝。
- **违反后果**：6 周 → 12 周 → 24 周漂移，回到原 160 人天陷阱，行为指标永远等不到。

### 🔴 4. Gate 是行为型 + 单位经济型，不通过就停回假设清单，不堆功能

- **依据**：≥ 10 用户 / ≥ 5 完成 / ≥ 3 回访（5/6 同意，Karpathy 不反对）+ 单条 < ¥15 / 支撑 ¥39 月（6/6 同意）。
- **可检验门**：Gate 不通过时强制走 §7 H1-H8 假设排查，禁止「再加一个功能看看」。
- **违反后果**：Buffett 视角的「institutional imperative」启动——不断加功能掩盖核心假设错误，把小生意做成大成本。

### 🔴 5. 学习闭环 / 资产复用必须在 Phase 1 就埋点（即使无法验证）

- **依据**：Ilya §2「如果每次创作都是独立 prompt chain 就没有学习」+ Buffett §3.1「如果每次使用都像第一次使用，就没有复利」+ Naval §2「工具会被替代，资产会复利」。
- **可检验门**：Phase 1 至少埋点「同一用户二次创作是否复用首次锚点 / 是否调整赛道描述 / 是否改写偏好被保留」；Gate 中「≥ 2 人开始沉淀锚点」从「试用」升级为「二次创作真的引用」。
- **违反后果**：Phase 3 才想做 learning loop 分析时无历史数据，护城河假设永远验证不了，Cascade 退回「昂贵 wrapper」。

---

## 7. 给策略阶段的总结

- **真共识 9 条（§3）= Phase 1 的不可移动地基。**
- **5 处分歧（§4）已被 `PHASED_PLAN` 用可辩护方式裁决**，但 D5（学习闭环时点）需在 Phase 1 埋点层面补强。
- **8 条 silent risks（§5）是 `PHASED_PLAN` 与评审共识的 gap**，应在进入 Phase 1 之前消化进 Gate / 埋点 / fixture 设计。
- **Top 5 红线（§6）是 Phase 1 的不可妥协项**，任一违反都使 6 周实验信号失效。

---

**单一最大缺口**：`PHASED_PLAN` 没有把「学习闭环 / 资产复用」在 Phase 1 就做成可埋点、可在 Gate 检验的硬指标——这是 Ilya 和 Buffett 都视为护城河的部分，被推迟到 Phase 3 才验证，而 Phase 1-2 没有相应数据采集计划，构成最严重的延迟反馈风险。
