# Cascade Phase 1 · UX Research
## 用户：30 岁宝妈，做辅食 + 育儿，抖音 / 小红书

**Date**: 2026-05-19
**Status**: UX Research v0（Phase 1 单一闭环验证用）
**Owner**: UX Researcher
**For**: Phase 1 (W3-W6) · 见 `PHASED_PLAN.md` §4
**Reading prereqs**: `PRODUCT_VISION.md` §2-§3 · `PHASED_PLAN.md` §4 · `CANVAS_DESIGN.md`（已 skim）

---

## 0. TL;DR · 三句话

1. **Phase 1 的闭环（URL → 浅分析 → 选赛道 → 改写 → 最小画布 → 草稿图 → 锚点复用 → 发布包）对宝妈来说有 7 个动作。她在第 1 个动作就可能放弃**——因为她现在没有"找一条抖音 URL 复制粘贴"的习惯，她是直接刷到觉得好就点收藏。
2. **"最小画布"对她不是简化，是新东西**——她从来没见过画布。FigJam / Miro / 节点连线都不在她的认知里。她见过的是抖音剪辑模板、相册、聊天框、表单。
3. **"锚点"这个词必须烧掉**——不只是名字，是整个交互。她要的是"上次那个我（或我家娃）的样子，这次还能用"——这是一句"用我之前的角色"按钮，不是一个左侧栏的资产库。

**核心结论**：Phase 1 必须用"对话流 + 模板填空" 伪装的画布，而不是真正的画布 UI。详见 §4。

## 1. User Story Map · Phase 1 单一闭环

下面我用宝妈第一人称（粗略）+ UX 观察一起写。每一步问 4 个问题：**她看到什么 · 她感觉什么 · 哪里会出错 · 什么会让她退出**。

### 1.1 步骤 0 · 她从哪里来到这个页面？

| 维度 | 内容 |
|---|---|
| 真实场景 | 朋友圈/小红书有人转发 → 她点开链接 → 在手机上看到 landing → 关掉 → 晚上 9 点孩子睡了，打开电脑（Phase 1 锁定桌面）|
| 她带着什么进来 | "听说这个能帮我做视频，比剪映还简单"；**没有任何 URL 准备好** |
| 风险 | 她以为打开就能用，不知道还要先去抖音找 URL 复制回来。这个**前置成本她不会预期到** |

**Phase 1 必须解决的事**：landing 的第一屏必须假设她**两手空空**，不能假设她已经粘了 URL。

---

### 1.2 步骤 1 · 粘贴一条爆款 URL

**她看到**：一个空输入框，提示词类似 "粘贴一条抖音/小红书爆款链接"。下面 5 张系统精选热点卡片。

**她感觉**：
- 输入框 → 困惑："我去哪里找 URL"
- 5 张精选卡片 → 这才是她真正会点的东西（她需要被推荐，不是被要求自己找）

**会出错的地方**：
- 她不知道怎么从抖音 App 复制 URL（抖音 App 的复制链接路径是 "分享 → 复制链接"，但她平时分享是直接转微信，没用过"复制链接"）
- 她在手机上找到了 URL 但电脑上没有打开微信/复制板同步
- 她粘进去了，但她粘的是小红书图文笔记（不是视频），系统会怎么处理？
- 她粘的是一条已经被删除/限流的视频

**让她退出的临界点**：
- 她在抖音翻了 3 分钟没找到复制链接选项 → "算了，明天再说"（再也不回来了）
- 提示出现"链接无效"、"分析失败"任何技术错误英文 → 直接关页面

**Phase 1 必须**：
- 默认让她**点 5 张精选卡片中的一张**，URL 输入框只是"高级用户备选"
- 5 张卡片必须**当天就是热的**（cron 每天更新一次起码），不然她看到 3 天前的话题立刻不信任
- 卡片下方有微小一行：📋 "怎么从抖音复制链接"（folded tutorial，1 张截图就够）

---

### 1.3 步骤 2 · 浅分析（看懂为什么火）

**她看到**：一张卡片，写着 `whyItHit` 和 `howToAdapt`。来自 toprador 的 8 维分析。

**她感觉**（这一步是 Phase 1 最有可能让她"哇"的地方）：
- 如果文案是 **"开头 3 秒抓眼球：用了一句吓你的话——'你家娃辅食这样吃直接住院'。然后 5 秒切到食材对比，让人想看下去。"** → 她会**点头**。这就是她平时刷抖音脑子里隐约觉得但说不出来的东西。
- 如果文案是 **"hook strength 8.7，叙事弧度 typeC，情绪曲线 ascending_climax"** → 她会**立刻关掉**。

**会出错的地方**：
- toprador 分析失败 / 字段缺失（Phase 0 已经在防这个，但 fallback 文案必须用人话）
- 分析时间 > 30 秒（她会以为卡了）
- 分析输出术语化（"strong-hook"、"narrative arc"、"engagement DNA"）

**让她退出的临界点**：
- 等待超过 30 秒没动静 → 她以为坏了
- 看到一堆英文术语 → 她觉得"这是给专业人士的，不适合我"

**Phase 1 必须**：
- 浅分析输出**必须人话化**：用 "开头怎么抓人 / 中间为什么不快进 / 结尾为什么忍不住点赞" 三段固定结构，不暴露 hook/arc/dna 这些词
- **必须有 streaming**——文案一行一行出来，让她觉得有人在思考（而不是空白等待）
- 分析失败时直接显示 fallback："这条视频的爆款公式没看清楚，换一条试试？" + 一键换其他热点卡片

---

### 1.4 步骤 3 · 选赛道（"宝妈辅食 / 3C / 本地生活"）

**她看到**：让她选一个赛道描述自己。

**她感觉**：
- "宝妈辅食" → 她知道这是她。✓
- "本地生活 / 3C 测评 / 美妆" → 她知道这不是她。✗
- 但她可能想做的是 **"宝妈辅食 + 自己出镜 + 偶尔搭一点亲子日常"**——这不是单一赛道，是组合。

**会出错的地方**：
- 选项太少 → 她找不到自己的（"我做的不算辅食，是育儿日常"）
- 选项太多 → 她选择困难
- 选完之后没有"我可以补充"的口子 → 她觉得被框住

**让她退出的临界点**：
- 必填字段、没默认值、要她做选择 → 这是宝妈最讨厌的（她每天做 100 个决策给娃，到这里不想再做）

**Phase 1 必须**：
- 赛道选项控制在 **6-8 个**（覆盖 80% 个人创作者垂类）
- **第一次进来时根据她选的那条 URL 自动预选**（"这条是辅食的，你也是吧？" + 单选按钮已勾上）—— 她只需要确认或换一个，不需要从零选
- 必须有一个 "其他（你自己说一句）" 口子，让她写一句话也能进下一步

---

### 1.5 步骤 4 · 系统给出改写后的脚本

**她看到**：一段文本，包含台词（dialogue）+ 镜头切分（"镜头 1：特写，5 秒"...）。

**她感觉**（这是另一个高光时刻）：
- 如果改写后的脚本**真的像她平时拍的样子**（用她日常说话方式 + 她家厨房元素 + 她娃的年龄段）→ 她会觉得"哇，这就是我要的"
- 如果改写后的脚本是**通用模板感**（"亲爱的宝妈们..."、"大家好我是 XX"）→ 她会觉得"这跟我没关系"

**会出错的地方**：
- LLM 改写出来的脚本太长 / 太短 / 不符合 60s 节奏
- 改写出来的对白不像她本人会说的话（口语 vs 书面、北方话 vs 南方话、有娃 vs 没娃的语气）
- 镜头切分粒度不对（8 个镜头一条 60s 视频对个人创作者太多，3-5 个更现实）
- 没有"我可以改"的口子，她只能 accept 或 reject 整个

**让她退出的临界点**：
- 脚本看着像 ChatGPT 写的（千篇一律的"今天给大家分享..."）→ 失望
- 完全没有她可以微调的地方 → "这不是我做的"

**Phase 1 必须**：
- 改写脚本输出**严格控制在 3-5 个镜头**（与 P1-4 目标一致），不要给她 8 个
- 每段对白下面**有"换一种说法"按钮**（agent regenerate 单段，不需要重 LLM 整条）
- 必须显示**"这条是从 X 改来的"**（trending source meta），保留信任锚点
- 默认对白风格设置一个简单 slider 或选项：**"像聊天 / 像朋友圈 / 像主播"** —— 三选一就够

---

### 1.6 步骤 5 · 最小画布（脚本节点 + 3-5 镜头节点 + 发布包节点）

⚠️ **这一步是 Phase 1 最大风险点**。详见 §4 心智模型分析。

**她看到**：一个画布，左侧资产库（角色 + 场景），中间脚本节点 + 镜头节点连成线，右下角发布包节点。

**她感觉**：
- 第一印象：**"这是什么？"**
- 节点 / 连线 / 拖拽 / 缩放 → 她**从来没见过**这种 UI（她见过的：抖音剪辑时间轴、相册、聊天框、表单、淘宝商品页）
- 看到左侧"角色锚点 / 场景锚点" → **"这两个词什么意思？"**

**会出错的地方**：
- 她不知道**镜头节点是干嘛的、点了会发生什么**
- 她拖错节点 / 点开节点后想关掉但不知道怎么关
- 画布缩放她不会用（trackpad pinch-to-zoom 对部分用户陌生）
- 节点连线被误碰断开，她不知道怎么恢复
- 节点上的状态图标（⌛ ☑ ⚠ ✏️）她看不懂

**让她退出的临界点**：
- **她看到画布的第一秒就觉得"这不是给我用的"** —— 即使她技术上能用，她心理上拒绝。这是 Jobs review 提到的 "iPod vs PC" 危险区。
- 她点了一个节点弹出一堆字段（prompt / provider / cost / status...）→ 她直接懵

**Phase 1 必须**（详见 §4 alternative）：
- **不要叫"画布"，不要展示节点连线**
- 给她看的应该是**"一条流水"**：上面是脚本（可编辑），下面是 3-5 张图片缩略图（每张下面一行台词），最下面是"导出"按钮
- 所有"节点 / 锚点 / DAG / 模式 / 配额"这些词不出现
- 资产库**塞进一个"我的素材"按钮**（折叠在侧边，不默认展开）

---

### 1.7 步骤 6 · 生成 3-5 张关键镜头草稿图

**她看到**：每个镜头位置先是 placeholder，然后 image gen 跑完，缩略图出现。

**她感觉**：
- 等待时如果有进度条 + "正在画镜头 2..." 这种 narration → 她觉得是有人在帮她
- 等待时如果是空白圈圈转 → 她以为卡了
- 出来的图如果**和她的赛道（辅食 + 宝妈 + 厨房）一致** → 满足
- 出来的图如果**风格诡异**（写实变赛博、人物比例失真、东亚脸生成成欧美脸、娃的年龄不对）→ 她觉得 "AI 还是不懂中国宝妈"

**会出错的地方**（这是最可预见的失败点）：
- Image gen 失败（apimart / gemini 超时 / 内容审核 / NSFW false positive）
- 风格漂移（每张图风格不一致——因为 Phase 1 是 image-grounded 但**不做三视图**，单图引用 → 一致性会差）
- 中国家庭/厨房元素生成不对（西式厨房、白人妈妈、不像中国的娃）
- 等待时间 > 90 秒（每张图）

**让她退出的临界点**：
- 第一张图出来就**人脸不像中国人** → "这不是给中国人用的"，立刻关掉
- 失败后只显示 "Generation failed" 英文 → 关掉
- 出来 5 张图，3 张人物不一致（同一个"妈妈"长得不一样）→ 她觉得"做出来的视频也是这样"

**Phase 1 必须**：
- Provider 默认必须是**对中文/亚洲面孔/中式生活元素友好的**（Gemini 2.5 / 即梦 / Apimart 选优）
- 镜头之间的**人物一致性**必须有保底机制：即使 Phase 1 不做三视图，也得让所有镜头**用同一张 character reference image**（单图引用 OK，但**必须是同一张**）
- 失败必须**人话化**："这张画失败了，要不要换个风格再画一次？" + 3 个明显选项（重试 / 换风格 / 跳过这张）

---

### 1.8 步骤 7 · 锚点复用（"上次的妈妈/厨房，这次还能用"）

详细心智模型分析见 §6。

**她看到**：在生成新一条 run 的时候，系统提醒"你之前做过的角色 / 场景可以直接用"。

**她感觉**：
- 如果说法是 **"用上次那个你"** → 立刻懂
- 如果说法是 **"复用角色锚点"** → 不懂
- 如果是默认勾上 + 让她确认 → 她会点一下确认（积极反应）
- 如果是空表让她从资产库选 → 她会忽略（找不到入口）

**会出错的地方**：
- 她**第一次创作**没有可复用的（这是她最容易卡住的点——她需要被告知"以后会越用越方便"）
- 她**做了两条 run**，但每条用的 prompt 描述不一样，系统识别不出"她想要的是同一个妈妈" → 锚点不能复用 → 她不知道这是为什么
- 锚点编辑入口在左侧栏 → 她找不到

**让她退出的临界点**：
- 她**第二次创作**进来发现没有任何"上次的"东西被记住 → 她以为这个产品没记忆 → 留存崩
- 她**手动建一个角色** 时间 > 2 分钟 → 她放弃，宁愿每次重做

**Phase 1 必须**：
- 第一条 run 时，**系统不要主动提锚点概念**，等她做完第一条之后再说："这次做的'妈妈'和'厨房'已经帮你存好了，下次还能用 →"
- 第二条 run 进来时，**默认 pre-fill** 上次的角色和场景，让她**点 X 删掉** 比让她**点 + 添加** 简单 10 倍
- "我的角色 / 我的场景" 这两个 list 必须有缩略图（不是文字列表），她能一眼认出"是上次的那个"

---

### 1.9 步骤 8 · 发布包导出

**她看到**：一个"复制到剪贴板"按钮，包含标题、标签、脚本、镜头图 URL。

**她感觉**：
- 如果导出的是**结构化卡片（标题在最上，标签 # 形式，脚本逐段，图片链接列表）**，并且明确说"已复制，去抖音粘贴吧" → 她懂
- 如果导出的是**一段 JSON / 一段长文本 / 一个 zip 文件** → 她不知道怎么用

**会出错的地方**：
- 复制到剪贴板，她去抖音 App 粘贴，发现抖音 App 的发布界面不接受"一长串文本"——她需要分别粘贴标题、标签、文案到不同字段 → 她操作 3 次才完成
- 镜头图 URL 是私链 / 过期 / 不可下载到手机
- 没有"下载所有素材到本地"按钮

**让她退出的临界点**：
- 导出的东西看不懂格式 → 她直接关页面
- 镜头图链接打不开 → 她觉得整个产品没用

**Phase 1 必须**：
- 导出**分块复制**：标题一个按钮、标签一个按钮、文案一个按钮、镜头图 zip 下载一个按钮（4 个按钮，不是一个)
- 给一个**短视频教程**"导出后怎么发抖音"（截图 + 箭头，30 秒看完）
- 镜头图必须支持**手机扫码下载**（电脑端创作 → 手机端发布的真实场景）

## 2. Top 5 Friction Points

按严重程度排序。Severity 三档：**showstopper（直接退出）** / **annoying（继续但不爽）** / **minor（无所谓）**。

### F1 · 第一步"粘贴 URL"前置成本被低估
- **Severity**: showstopper
- **发生在**：步骤 1
- **为什么严重**：宝妈打开 landing 时**手里没有 URL**。她不知道从抖音怎么复制链接，不知道在哪里粘贴。她会以为这个产品是 "我说一句话就能出视频"，而不是 "我要先去抖音找一条爆款"。
- **UX mitigation**：
  - landing 默认状态**强推 5 张"今日精选热点"卡片**，URL 输入框 demote 成"或者粘贴你看到的爆款链接 →" 一行小字
  - 第一次进来的用户**强制走"选一张精选卡片"路径**，她做完第一条 run 之后**再解锁**"自己粘贴 URL" 的玩法
  - 卡片下方一行：📋 "📱 从抖音复制链接 (3 步)" → 折叠教程，点开是一张截图 3 个箭头

### F2 · "画布 + 节点 + 锚点"对宝妈是外星语言
- **Severity**: showstopper
- **发生在**：步骤 5
- **为什么严重**：她从未在任何产品里用过节点画布（FigJam / Miro / n8n / Notion DB 这些都不在她的生活里）。即使 Phase 1 是"最小画布"，节点 + 连线本身就是一个新心智模型。她在画布上待 10 秒内会做出"这不是给我用的"的判断。
- **UX mitigation**：
  - **不要叫画布**，叫"创作台" / "我的视频"
  - 视觉上**用 vertical stack（垂直流）** 代替 horizontal canvas：脚本卡片在上、镜头 3-5 张图片排成一列在中、发布按钮在下
  - **不暴露节点连线**——节点是数据模型层概念，UI 层应该是"卡片"
  - 状态图标（⌛ ☑ ⚠）改成**人话标签**："正在画 / 画好了 / 画失败了"
  - 详细 alternative 提案见 §4

### F3 · 浅分析输出"专业感"会吓退她
- **Severity**: showstopper
- **发生在**：步骤 2
- **为什么严重**：toprador 8 维分析的字段名（hook_strength / narrative_arc / engagement_dna / emotional_curve）如果**直接渲染**到 UI，宝妈会立刻觉得"这是专业 MCN 用的工具"。她要的是"为什么这条视频火"的人话解释。
- **UX mitigation**：
  - 浅分析输出**强制走"三段式人话模板"**：开头怎么抓人 / 中间为什么不快进 / 结尾为什么忍不住点赞
  - **绝不暴露字段名**——后端 8 维分析 → 前端 3 段叙事
  - 文案模板必须由**真实创作者审过**（不是工程师写）
  - **streaming 输出**——一句一句出来，比一次性 dump 更有"被服务"感

### F4 · 镜头草稿图人物一致性 + 中国元素准确性
- **Severity**: showstopper（如果失败）/ annoying（如果勉强）
- **发生在**：步骤 6
- **为什么严重**：Phase 1 明确**不做三视图锚点**（只单图引用，见 P1-6）。这意味着 3-5 张草稿图之间的人物一致性靠 prompt + 单 reference image 维持。实际效果会**飘**。宝妈对"妈妈这张脸"的识别比对镜头数量、镜头节奏更敏感——人脸一变，她觉得"做的不是我"。中国家庭元素（厨房、餐具、娃的脸）如果生成出欧美/日式，她会立刻不信任。
- **UX mitigation**：
  - **强制所有镜头共享 1 张 character reference**（同一张图作为 image-grounded video / image gen 的种子）
  - **Provider 路由**：人物镜头必须走对中文/亚洲面孔友好的模型（Gemini 2.5 / 即梦 / Apimart 选优，跑 fixtures 验证）
  - Phase 0 的 fixture corpus 应**至少包含 5 条母婴/育儿类视频**，验证 image gen 在这个垂类的视觉一致性
  - 失败处理：**让她一键"重画这一张"**（不是重做整条），费用 < ¥1 / 次
  - 在镜头图下面默认显示一行 micro-copy："不像你？告诉我们怎么调"

### F5 · 第一条 run 完成后 → 第二条 run "锚点复用"入口找不到
- **Severity**: annoying（但累计影响留存）
- **发生在**：步骤 7（实际触发在第二次回访）
- **为什么严重**：Phase 1 Gate 第 3 项硬指标是"一周内回来做第二条 ≥ 3 人"。但**她回来的时候，如果发现"上次的妈妈、上次的厨房"找不到，她会以为产品没记住她**。锚点的"复利"价值在 UI 上必须立刻可见，不能藏在左侧栏。
- **UX mitigation**：
  - 第一条 run 完成后的**完成屏**显示：" 你做的'妈妈'和'厨房'已经存好了，下次还能用 → "
  - 第二条 run 进入第 4 步（脚本生成）时，**默认 pre-fill 上次的角色和场景**，并显示"用上次的（你创作过的）"标签，让她**点 X 删掉**比让她从空白添加简单
  - **首页**显示"上次创作的角色 / 场景"缩略图区块（不只是"项目"列表）
  - 详细命名建议见 §6

## 3. 冷启动 60 秒 · 第一屏

宝妈刚打开 landing。她要在 60 秒内决定"我用不用"。

她**不会**：
- 看说明文档
- 看 Logo / hero 文案 / 价值主张那一堆
- 注册账号（如果立刻要她填邮箱密码 → 关）

她**会**（如果做对）：
- 看到一个跟她生活相关的视觉（不是科技感的渲染图）
- 看到一个**已经会用的动作**：点一下、选一个
- 看到"3 分钟做完一条"这种**具体承诺**（不是"AI 视频创作平台"）

### 3.1 第一屏的"一句话 + 一个元素"提案

按转化潜力排序。

#### 变体 A · 帮她选（推荐 · 最低门槛）
```
顶部一句话：
"看到刷屏的视频想做一条同款？挑一张开始 ↓"

下方主元素：
3 张大卡片（不是 5，避免选择困难症），每张写：
- 缩略图（真实抖音封面）
- 标题（"5 点起床实录" "辅食吃出住院" "二宝妈崩溃瞬间"）
- 一行人话标签（"7.8 万赞 · 适合宝妈赛道"）
- 大按钮："用这条做一个我的"
```
**为什么**：完全消除"先去找 URL"的前置成本。她做的事是"挑封面"——这是她每天在小红书做了 1000 次的动作。

#### 变体 B · 帮她想（次推荐 · 适合品牌 storytelling）
```
顶部一句话：
"今天宝妈在抖音最爱看的，是这 3 条 ↓"

下方主元素：
和变体 A 一样的 3 张卡片，但顶部加一行：
"今天 12:00 更新 · 已过滤掉无关赛道"
```
**为什么**：加入"为她筛"的服务感，强化"系统替我做了一些活儿"的信号。但相比 A 多一句话，加载首屏要慢一秒。

#### 变体 C · 让她说（不推荐 · 当作 fallback）
```
顶部一句话：
"用一句话说你想做的视频，我帮你找参考"

下方主元素：
一个大输入框，下面 3 个 chip：
[#自律早起]  [#辅食教程]  [#二宝日常]
```
**为什么不推荐**：宝妈现在脑子里没有"想做什么视频"这种 query intent。她是**被刷到 → 模仿** 模式，不是 **想做 → 搜索** 模式。这个变体把决策成本推给她。

### 3.2 60 秒倒计时她要完成的事

| 秒数 | 她做什么 | 系统反馈 |
|---|---|---|
| 0-3 秒 | 扫一眼整屏 | 视觉要"暖"——人 + 厨房 + 娃，不是 AI/科技感 |
| 3-10 秒 | 看 3 张卡片标题 | 标题必须**当天热**、必须**像她赛道**、必须**人话** |
| 10-20 秒 | 点其中一张的"用这条做一个我的" | 一键直接进入步骤 2（浅分析），**不要要求注册** |
| 20-50 秒 | 看浅分析动起来（streaming 出文字） | 系统在"思考"的体感强，文字逐句出 |
| 50-60 秒 | 看到 "选你的赛道"，第一个选项已预选"宝妈辅食" | 她点确认进下一步 |

**关键决策**：注册账号**推迟到她已经看到改写脚本之后**（步骤 4 之后）。让她**先尝到甜头**再要 commitment。

## 4. 心智模型风险 · 画布 UI 还是错的吗？

> Jobs review 的核心："iPod, not PC. 严格简单。"

### 4.1 直接结论

**画布 UI 在 Phase 1 对这个 persona 是错的**。

不是"画布的数据模型"错——DAG / nodes / anchors 在底层是对的（Phase 2/3 价值会显现）。
**错的是"把画布作为 UI 主体暴露给宝妈"**。

她的真实心智模型不是"我在导演一个剧组"。她的真实心智模型是：

> **"我刷到一条好的，我也想做一个差不多的，但是是我自己版本的。"**

这个心智里没有"剧组" / "锚点" / "节点" / "DAG" / "宫格图"。
有的是："参考视频" + "我的视频" + "妈妈本人" + "厨房" + "娃"。

### 4.2 PRODUCT_VISION 里的画布愿景是给谁的？

回到 `PRODUCT_VISION.md` §2 看用户层级：

| 用户层 | 画布是否合适 |
|---|---|
| 个人创作者 / 自媒体（MVP 锁定） | ❌ 不合适 — 她的认知里没有节点编辑器 |
| MCN / 工作室（V1.5） | ✓ 合适 — 他们要管 5+ IP 矩阵，需要可视拓扑 |
| 品牌甲方（V2+） | ✓ 合适 — 他们要审稿流 + 版本管理 |

**所以画布是 V1.5+ 的差异化资产，不是 Phase 1 的产品 UI**。
Phase 1 验证的是"热点 → 改写 → 发布"闭环对个人创作者有用，不是"画布对个人创作者有用"。

### 4.3 Phase 1 应该长什么样 · "向下兼容画布"的卡片流

数据模型保留 Cascade 画布 DAG（不动）。UI 层换成宝妈认得的形态：

```
┌───────────────────────────────────────────────────┐
│ ← 返回   我做的：宝妈辅食版「5 点起床实录」          │
│                                       3 分 12 秒 ✓│
├───────────────────────────────────────────────────┤
│                                                   │
│ 📝 我的文案                                        │
│ ┌─────────────────────────────────────────────┐ │
│ │ 5 点的厨房就我和小娃，闹钟一响...               │ │
│ │ [换种说法]                                    │ │
│ └─────────────────────────────────────────────┘ │
│                                                   │
│ 🎬 我的镜头 (3-5 张图)                            │
│                                                   │
│ ┌──────────┐  ① 卧室闹钟特写 · 3 秒              │
│ │  [图片]  │  "5 点 30 分..."                    │
│ │   ✓      │  [换张图]                            │
│ └──────────┘                                      │
│                                                   │
│ ┌──────────┐  ② 厨房早晨远景 · 4 秒              │
│ │  [图片]  │  "我轻轻起身..."                    │
│ │   ⏳     │  [等待画完]                          │
│ └──────────┘                                      │
│                                                   │
│ ┌──────────┐  ③ 备辅食特写 · 5 秒                │
│ │  [图片]  │  "今天给小娃做..."                  │
│ │   ✓      │  [换张图]                            │
│ └──────────┘                                      │
│                                                   │
│ ─────────────────────────────────────             │
│                                                   │
│ 这条用了：                                         │
│ 👤 妈妈本人（已存）  🏠 我家厨房（已存）           │
│ [看看我的角色和场景]                               │
│                                                   │
│ ─────────────────────────────────────             │
│                                                   │
│      [📋 复制发布到抖音]    [💾 下载所有图片]       │
│                                                   │
└───────────────────────────────────────────────────┘
```

**这个 UI 的本质是一个"卡片流"，对应的底层数据还是 DAG**。
节点 → UI 上是卡片
连线 → UI 上是垂直顺序
锚点 → UI 上是"妈妈本人" + "我家厨房"

这样：
- 后端不用改（数据模型保留 Phase 2/3 价值）
- 宝妈看到的是"我的视频"，不是"画布"
- Phase 2 加 MCN 时，再加一个"切到完整画布"的开关（V1.5 才暴露）

### 4.4 但保留一个画布 UI 隐藏入口

Phase 1 用户中如果有**早期种子用户是 MCN / 工作室人员**（创始人邀请的），给他们一个 **"切到专业视图"** 入口，看到完整 React Flow。
这样：
- 个人创作者看不到画布（默认 hide）
- MCN 看到画布（差异化锁点验证 Phase 2/3）
- 数据沉淀一份

## 5. 测试什么 · 10 人访谈脚本

按 PHASED_PLAN §4.4 Gate：**真实创作者 10 人试用 + 完成第一条 ≥ 5 人 + 一周内回来做第二条 ≥ 3 人**。

### 5.1 招募标准
- 30-40 岁
- 女性
- 抖音 / 小红书有自己账号（≥ 50 粉丝即可，不必爆款）
- 内容方向：母婴 / 辅食 / 育儿 / 家庭日常（5 个）+ 其他个人垂类（5 个，作为对照）
- 单条视频今天**花 ≥ 2 小时**做（验证 pain point）
- 愿意做 **45 分钟访谈 + 1 周后 15 分钟回访**

### 5.2 访谈脚本（5 个问题 + 1 个任务）

**Setup (5 min)**：
"今天想请你帮我看一个我们做的早期工具，不是评测你，是评测我们。你怎么用都行，怎么想的就说出来。"

**Phase A · 让她裸跑（25 min）**：

> "你打开这个网页，假装是周末晚上想做一条新视频。能不能边做边说一下你看到什么、在想什么？"

**只观察，不引导**。记录：
- 她在哪一步卡住超过 10 秒
- 她说出口的疑问（"这是啥"、"我点哪里"、"这个图怎么换"）
- 她什么时候**笑** 或 **点头**（高光时刻）
- 她什么时候**皱眉** 或 **沉默** （失败信号）
- 她**完成第一条的总时长**

**Phase B · 5 个问题（15 min）**：

**Q1**（开放）：
"刚才整个流程，最让你觉得'这东西能帮我'的是哪一步？为什么？"
→ 验证 H2、H3（浅分析 + 改写脚本的核心价值）

**Q2**（开放）：
"刚才最让你想关掉页面的是哪一步？为什么？"
→ 验证 §2 的 Top 5 friction（看实际 vs 假设）

**Q3**（具体）：
"这 3-5 张镜头图，你觉得能拿去发吗？你愿意贴上抖音吗？"
→ 验证 H5（生成质量足以发布）
追问："如果只 3 张图，剩下你自己拍，可以吗？"

**Q4**（具体）：
"你刚才用过的'妈妈本人'和'我家厨房'这两个，下次做新视频的时候，你想直接用还是想重新生成？"
→ 验证 H4（锚点复用是用户能感知并主动使用的）
追问："如果系统记住了，你觉得这是加分项还是必备的？"

**Q5**（具体）：
"如果这个工具按月收钱，¥39/月能用 30 条，你会用吗？什么样的话你会真的付费？"
→ 验证 H6（¥39 模型 + 付费 trigger）
**追问关键**："你今天做完这一条，觉得**省了多少时间**？省那些时间值不值这个钱？"

**Phase C · 一周后回访（15 min · 7 天后做）**：

只问 2 件事：
- "你这周回来用了吗？没用为啥？用了几次？"
- "回来用的时候，'上次的妈妈、上次的厨房'你找到了吗？有没有用上？"

### 5.3 必须出现的 3 个行为（Gate）

如果这 3 个行为在 10 人里出现不达标，Phase 1 不通过。

| 行为 | 通过线 | 衡量方式 |
|---|---|---|
| **B1 · 完成第一条** | ≥ 5/10 人在 30 分钟内完成（从打开 landing 到拿到发布包） | 时间戳 + screen recording |
| **B2 · 一周内回来做第二条** | ≥ 3/10 人主动回访（不发邮件提醒） | login analytics |
| **B3 · 第二条主动用了之前的角色或场景** | ≥ 2/10 人在第二条 run 里复用了第一条留下的锚点 | event: anchor_reused |

不达标 → 回 PHASED_PLAN §7 假设清单 H1-H8 查哪个假设错了，**不堆功能**。

### 5.4 不要测的事（明确）

- 不要测"她喜不喜欢画布"——这是设计问题，不是用户问题。如果她用得不爽，就改 UI，不是问她
- 不要测"她愿意给多少钱"——单凭意向调查不准。看实际付费转化
- 不要做 SUS / NPS / 标准量表——10 人样本太小，量表无意义
- 不要让她回答"你最希望增加什么功能"——这是反模式（Henry Ford 的 faster horse）

## 6. "锚点"用人话怎么讲

### 6.1 "锚点 (Anchor)" 这个词的命运

**直接结论**：这个词**永远不要出现在 UI 文案里**。
保留在代码 / schema / 内部文档（Cascade、CANVAS_DESIGN.md 沿用）。
对外只用人话。

### 6.2 角色锚点 (Character Anchor) → 人话

| 内部叫法 | 宝妈面前叫什么 |
|---|---|
| character anchor | **我的角色** / **"妈妈本人"** / **"我家小娃"** |
| 跨 run 复用 | **"上次的妈妈，这次还能用"** / **"用上次的你"** |
| 三视图 (front / side / back) | （Phase 1 不做，Phase 2 再说：**"正面、侧面、背面三张"**） |
| 风格定稿板 | **"你想要她长什么样子"** |
| confirmed | **"已经定下来"** |
| reviewing | **"等你看看像不像"** |

**最佳一句话引导**（在第一次锚点出现时）：

> 🎬 我们帮你画了**这个故事里的"妈妈"和"娃"** —— 觉得像就保留，下次做新视频还能用。

### 6.3 场景锚点 (Scene Anchor) → 人话

| 内部叫法 | 宝妈面前叫什么 |
|---|---|
| scene anchor | **我的场景** / **"我家厨房"** / **"卧室早晨"** |
| 关键元素 | **"里面有啥"**（"木质台面、窗外的光、小娃椅"） |
| 跨 run 复用 | **"以后做厨房的视频，都用这个"** |
| grid / 宫格图 | （Phase 1 不做） |

**最佳一句话引导**：

> 🏠 这是你视频里的"**厨房**" —— 觉得像就保留，做新视频时候直接拿来用。

### 6.4 锚点 "复用" 这个动作的 UI 文案

| 场景 | 错的叫法 | 对的叫法 |
|---|---|---|
| 第一次创作完成后 | "你的锚点已保存到资产库" | **"你做的'妈妈'和'厨房'已经存好了，下次还能用 →"** |
| 第二次创作时的 pre-fill | "已加载历史 anchor，请确认" | **"用你上次的妈妈和厨房？" [是] [换一个]** |
| 资产库入口 | "Asset Library" / "我的资产" | **"我的角色 ·我的场景"**（两个 tab） |
| 资产卡片上的状态 | "Confirmed · Used 3 times" | **"做过 3 次"** |
| 编辑锚点 | "Edit anchor properties" | **"调整一下她"** / **"换个样子"** |

### 6.5 锚点的"复利感"必须在 UI 上看见

宝妈的留存关键：她每次回来都觉得**"上次的东西还在，越用越省事"**。

**Phase 1 必做的两个微小 UI 信号**：

1. **首页显示"我的角色 · 我的场景"**（每个 tab 一行，缩略图横滚）
   - 第一次回访时她看到的是"上次的妈妈、上次的厨房" 已经在那里
   - **比"我的项目"列表更重要**（她不记得项目名，但她记得"我家妈妈那个"）

2. **完成屏 + 下次入口闭环**：
   - 完成屏文案：**"你做的'妈妈'和'厨房'已经存好了，下次做新视频可以直接用 →"**
   - 这一句把"复利"的承诺**显式说出来**，否则她不知道这个产品在帮她攒东西。

### 6.6 锚点术语 cheatsheet（开发对照）

| 后端 / schema | UI 文案 | 备注 |
|---|---|---|
| `anchor.type='character'` | 我的角色 | 永远不叫"character" |
| `anchor.type='scene'` | 我的场景 | 永远不叫"scene" |
| `anchor.type='grid'` | （Phase 1 不暴露） | 留 Phase 2/3 |
| `anchor.status='reviewing'` | "等你看看像不像" | 不叫 "Reviewing" |
| `anchor.status='confirmed'` | "已经定下来" / "保存好了" | 不叫 "Confirmed" |
| `anchor.usage_count` | "做过 X 次" | 不叫 "Used X times" |
| `views_json.front/side/back` | (Phase 2) 正面 / 侧面 / 背面 | 不叫 "Views" |
| `anchor_usage` 表 | （内部） | 用户不感知 |

## 7. 总结 + Phase 1 设计 do/don't

### 7.1 Do（必做）

- **landing 默认主推 3-5 张"今日精选热点卡片"**，URL 输入框降级
- **浅分析输出固定 3 段人话叙事**（开头怎么抓人 / 中间为什么不快进 / 结尾为什么忍不住点赞）
- **赛道自动预选**（根据 URL 推断），让她确认而不是从空白选
- **改写脚本严格 3-5 镜头**，每段对白下面有"换种说法"按钮
- **"画布"在 UI 上叫"我的视频"**，垂直卡片流，不暴露节点连线
- **所有镜头共享 1 张 character reference**，保证人物一致性
- **完成屏明确说"你做的'妈妈'和'厨房'已存好"**，让锚点价值显式
- **第二条 run 默认 pre-fill 上次角色 + 场景**，让删除比添加简单
- **首页显示"我的角色 · 我的场景"缩略图**，让回访能立刻看见沉淀
- **发布包分块复制（标题 / 标签 / 文案 / 图片下载）4 个按钮**
- **失败必须人话化**（"这张画失败了" 而不是 "Generation failed"）

### 7.2 Don't（明确反对）

- ❌ 不要在 UI 暴露"画布 / 节点 / 锚点 / DAG / hook / arc / dna / provider" 等任何术语
- ❌ 不要让她第一次进来就要粘 URL（强推卡片优先）
- ❌ 不要让她注册才能开始（推到改写脚本之后）
- ❌ 不要 8 个镜头（3-5 镜头，与 P1-4 一致）
- ❌ 不要让她在左侧资产库里"找"角色 / 场景（默认 pre-fill）
- ❌ 不要给她标准量表问卷（SUS / NPS 在 10 人样本无意义）
- ❌ 不要问她"你希望加什么功能"（faster horse 反模式）
- ❌ 不要在 Phase 1 暴露三档模式（闪电 / 标准 / 精修），只一个流
- ❌ 不要做完整 React Flow 画布的 UI（保留底层数据模型即可）

## 8. 给工程 / 设计的最小约束清单

```
landing 首屏              -> 3-5 张精选热点卡片 + URL 备选
浅分析 UI                 -> streaming 3 段人话，不暴露 8 维字段
赛道选择                  -> 系统预选 + "其他" 兜底，6-8 选项
改写脚本                  -> 3-5 镜头硬限，每段对白可"换种说法"
最小画布                  -> 实际渲染为 vertical card stack，不显节点连线
镜头草稿图                -> 1 张 character ref 跨所有镜头，provider 默认 Gemini/Apimart 选优
锚点                      -> UI 文案永远叫"我的角色 / 我的场景"，永远不出现"锚点"
锚点复用                  -> 第二条 run 默认 pre-fill，UI 上用"用上次的妈妈和厨房？"
发布包                    -> 分块复制（4 按钮）+ 镜头图扫码下载
失败处置                  -> 人话失败 + 3 选 1 兜底（重画 / 换风格 / 跳过）
```
