## 1. 品牌一句话（公开版）

### 1.1 锁定的对外句

> **「Cascade · 看见的爆款，做成你的版本。」**

### 1.2 为什么是这句话而不是别的

| 候选 | 为什么不用 |
|---|---|
| 「面向中文创作者的 AI 视频创作平台」 | 与即梦/可灵/剪映重合，零差异化（trend §6.4） |
| 「把别人的爆款变成你的版本的中文创作者工作系统」 | 内部锚句正确，但口语太长、"工作系统"太工程师（UX §6） |
| 「30 分钟产出一条爆款」 | 承诺无法兑现，Phase 1 还要回剪映/抖音补真人镜头（trend §3.3） |
| 「锚点为主角的 AI 视频画布」 | 工程师视角，宝妈 10 秒内退出（UX §4） |

**「看见的爆款，做成你的版本」**：
- "**看见的**" — 她每天在抖音/小红书刷到的（trend §3.1 真实工作流）
- "**做成**" — 不承诺全自动，留人参与空间（reviewer §3 关键节点人类兜底）
- "**你的版本**" — 个人化、不是抄袭、不是模板（差异化锁点 trend §6.2）

### 1.3 landing-page hero 必须说的那一句

英雄区第一屏第一行（H1）**只能是**：

> **看到刷屏的视频，想做一条自己的？挑一张开始 ↓**

理由：
- 直接对接 UX §3.1 变体 A（最低门槛入口）
- 不出现 AI / 平台 / 工具 / 智能 / 视频生成 等任何术语
- 第一动作（"挑一张"）是宝妈每天在小红书做 1000 次的肌肉记忆
- 紧接的 3 张热点卡片是真正的 hero 元素，文字仅为引子

## 2. 品牌 3 关键词（每个标签、每条文案、每个按钮都要回答）

### 2.1 三关键词

> **看懂 / 接手 / 留住**

每个 UI 文案在被批准前必须先回答：「这句话是在让用户**看懂**什么、**接手**什么、还是**留住**什么？」如果三个都不是 → 删掉这句话。

### 2.2 三关键词逐条辩护

#### 关键词 1 · **看懂**（Make it readable）

**含义**：把抖音爆款里"她隐约觉得但说不出来的东西"用人话说出来。

**对应能力**：
- 浅分析 whyItHit / howToAdapt（PHASED_PLAN P1-2）
- toprador 8 维 → 前端 3 段叙事的转译层（UX §1.3 F3）
- hotspot 算法分数 → 人话标签（"这条好做" / "今天还在涨")

**为什么是它而不是别的**：
- 即梦/可灵/剪映**全都不解释为什么火**（trend §1.1-§1.4 全部"不解决"列表第 2 项）
- 这是 Cascade 的**第一个"哇"时刻**（UX §1.3）—— 用户付费意愿的真起点
- 不是"AI 黑科技"，是"理解你的处境"——精准踩在"温度 + 理解"蓝海

**反例**：「8 维爆款分析引擎」← 是"看懂"的反义词

#### 关键词 2 · **接手**（Pick up where she left off）

**含义**：不要求用户从零开始。她带着一条爆款进来，系统**接住**这条爆款，给她下一步。

**对应能力**：
- URL 输入 → 一键浅分析 → 一键改写 → 一键出草稿（P1-1 → P1-5）
- 赛道自动预选（UX §1.4）
- 失败时主动提 3 选 1 兜底（UX §1.7 / reviewer §3.9）
- 完成后系统主动说"下次还能用 →"（UX §6.5）

**为什么是它而不是别的**：
- 宝妈"每天给娃做 100 个决策，到这里不想再做"（UX §1.4）
- 剪映/即梦把所有选择推给用户（参数、provider、prompt 模板）；Cascade 把选择留在内部，把动作交给用户
- "接手"是**温度**的具体表达——不是冷的工具，是"我懂你累了，我替你走一步"

**反例**：「请选择您的视频生成参数」← 是"接手"的反义词

#### 关键词 3 · **留住**（Keep it for next time）

**含义**：她做过的"妈妈"、"厨房"、"娃"，系统**记住**了。下次回来不用从零。

**对应能力**：
- 锚点跨 run 复用（P1-6）
- 完成屏的"已经存好了，下次还能用 →"（UX §6.5）
- 第二条 run pre-fill 上次角色 + 场景（UX §1.8）
- 首页"我的角色 · 我的场景"缩略图区块（UX §6.5）

**为什么是它而不是别的**：
- Buffett §3.1 + Ilya §2 + Naval §2 都把"复利"列为唯一真护城河（reviewer §3 / §6 红线 5）
- 一周回访 ≥ 3/10 人的 Gate 直接取决于"她回来时是否看见自己的沉淀"（UX §2 F5）
- "留住"是对**承诺**的兑现——证明 Cascade 不是又一个一次性工具

**反例**：「资产库管理 · 角色锚点已存档」← 是"留住"的反义词

### 2.3 三关键词的层级与触发时机

| 阶段 | 主关键词 | 体现位置 |
|---|---|---|
| Landing → 首条 run 步骤 1-2 | **看懂** | 热点卡片标题 + 浅分析三段叙事 |
| 首条 run 步骤 3-7 | **接手** | 赛道预选 + 改写"换种说法" + 失败 3 选 1 |
| 首条 run 步骤 8 + 第二条 run | **留住** | 完成屏 + 二次回访 pre-fill |

## 3. Tone of Voice · 5 do's + 5 don'ts

### 3.1 Do（必做）

1. **像朋友（不是客服，不是 AI）** — 用"你 / 你的"，不用"您 / 用户"
   ✓ `你做的这条已经画好了，要不要看一眼？`
   ✗ `尊敬的用户，您的视频生成任务已完成，请查收。`

2. **用动作动词，不用名词** — 句子里第一个有重量的词必须是动词
   ✓ `挑一张开始 →`
   ✓ `换种说法` / `换张图` / `存好了`
   ✗ `创作入口` / `内容生成` / `素材资产管理`

3. **承认有人在干活，不装神** — Karpathy reviewer §2.3 "Iron Man suit, 不装神"
   ✓ `我们帮你画了"妈妈"和"厨房" — 觉得像就保留`
   ✓ `这张画失败了，要不要换个风格再试？`
   ✗ `AI 智能生成中` / `算法正在优化你的创意`

4. **数字要小、单位要日常** — 用"3 张图 / 30 分钟 / 1 次"代替"3 frames / 30 mins / 1 generation"
   ✓ `3-5 张关键镜头，30 分钟出第一稿`
   ✗ `5-frame storyboard generation in under 30 min runtime`

5. **失败时把下一步直接给她（不只是道歉）** — Jobs §9 通过线 100%
   ✓ `这张画失败了 [换个风格再画] [跳过这张] [整段重做]`
   ✗ `生成失败，请稍后重试` / `Error: provider timeout`

### 3.2 Don't（明确反对）

1. **不出现任何"AI 黑科技"词** — 这是即梦/可灵/剪映已经占满的红海词
   ✗ `AI 智能改写引擎` / `多模态大模型` / `深度神经网络驱动` / `算法加持`
   ✓ `帮你把这条改成你的版本`

2. **不出现任何工程术语** — 即使懂的人也讨厌读
   ✗ `节点` / `锚点` / `DAG` / `Agent` / `画布` / `Provider` / `Prompt` / `Pipeline` / `Schema`
   ✓ 用 §4 术语对照表的右栏

3. **不用"赋能 / 助力 / 一站式 / 全方位"等公关黑话**
   ✗ `赋能创作者，一站式打造爆款`
   ✓ `看到刷屏的视频，做一条自己的`

4. **不出现英文术语（除了平台名）** — 中文产品就是中文产品
   ✗ `Hook strength: 8.7` / `Generation failed` / `Asset Library` / `Anchor`
   ✓ 中文 + 必要时保留"抖音 / 小红书 / 剪映"作为平台专名

5. **不用感叹号、不用 emoji 堆砌、不用"震惊体"**
   ✗ `🎉🎉一键生成爆款视频！！！太强了吧！！！`
   ✓ `挑一张开始 →`（一个 emoji + 一个箭头是上限）

### 3.3 一条额外硬规则

> **每条用户可见的句子在 commit 前自问**：把这句话给一个 30 岁宝妈看，她能在 3 秒内说出"这是给我做什么的"吗？做不到就重写。

## 4. 术语对照表（开发对照 · 工程语 → 用户语）

> **左栏是 schema / 代码 / 内部文档可保留的词**。右栏是 UI / 文案 / loading / failure / 营销帖**唯一允许出现的词**。任何 PR 出现左栏词在 UI 文本里 → 驳回。

### 4.1 产品形态层

| 工程语 | 用户语 |
|---|---|
| 画布 / Canvas | **创作台** / **我的视频** |
| 节点 / Node | **视频里的一段** / **一个镜头** |
| 连线 / Edge | （不可见 — UI 用垂直顺序代替） |
| DAG / 流程图 | （不暴露） |
| 锚点 / Anchor | **角色** / **场景** /（最高层抽象）**你存下来的东西** |
| Character Anchor | **我的角色** / 具体名（"妈妈本人"、"小娃"） |
| Scene Anchor | **我的场景** / 具体名（"我家厨房"、"卧室早晨"） |
| Grid Anchor / 宫格图 | （Phase 1 不暴露） |
| Views (front/side/back) | （Phase 1 不暴露；Phase 2 → 正面 / 侧面 / 背面） |
| Project / 项目 | **我的视频**（一条 = 一条视频，项目层 Phase 1 不暴露） |
| Run / 一次创作 | **一条**（"做一条" / "上一条" / "下次再做一条"） |

### 4.2 分析与生成层

| 工程语 | 用户语 |
|---|---|
| 浅分析 / Shallow Analysis | **它为什么火** |
| 深分析 / Deep Analysis / toprador full | **看仔细一点**（Pro 解锁，Phase 1 不暴露付费） |
| Hook strength / 钩子强度 | **开头怎么抓人** |
| Narrative arc / 叙事弧 | **中间为什么不快进** |
| Engagement DNA / 互动 DNA | **结尾为什么忍不住点赞** |
| viral_analysis | （后端字段，UI 永不出现） |
| Replicability score | **这条好做的程度** |
| Trending score / hotspot 评分 | **今天还在涨** / **够不够热** |
| 飙升话题 / Surging | **今天突然在火的** |
| 蓝海/红海 / Niche map | （Phase 1 不暴露） |
| Generation / 生成 | **画** / **做**（"画一张图" / "做一段") |
| Image-grounded video | （不暴露） |
| Provider / 模型供应商 | （不暴露） |
| Prompt | **怎么描述的**（如果一定要暴露，叫"画面描述"） |
| Streaming output | （不暴露 — 用 loading copy 体现"正在思考") |

### 4.3 流程与状态层

| 工程语 | 用户语 |
|---|---|
| Reviewing / 待审核 | **等你看看像不像** |
| Confirmed | **保存好了** / **已经定下来** |
| Failed | **没画出来** / **这张失败了** |
| Pending / Queued | **正在画** / **等一下** |
| Regenerate | **再画一次** / **换张图** |
| Retry with style | **换个风格再试** |
| Skip | **跳过这张** |
| Restart from scratch | **整段重做** |
| Rewrite paragraph | **换种说法** |
| 闸门 / Gate | （不暴露 — 用户感受不到流程闸门） |
| Agent 弹起 / Agent trigger | （不暴露 — Phase 1 砍掉所有 Agent UI，reviewer §3） |
| Quota / 配额 | **本月还能做几条**（数字直接显示，不要"剩余配额"） |
| usage_count | **做过 X 次** |

### 4.4 商业化与系统层

| 工程语 | 用户语 |
|---|---|
| Free / Pro / Team Tier | （Phase 1 只有"免费 3 条" + "每月 30 条 ¥39"，不暴露 tier 名词） |
| planTier='pro' | （内部字段，UI 永不出现） |
| Tenant / Workspace | （Phase 1 不暴露） |
| OAuth / 一键发布 | **复制到剪贴板** / **下载图片**（Phase 1 不接 OAuth，reviewer §3） |
| Publish package / 发布包 | **发布素材**（标题 / 标签 / 文案 / 图片 4 个按钮，UX §1.9） |

## 5. Visual / UI Guardrails · Phase 1 only

> **约束**：1 个工程师，Tailwind + Radix，无设计师。所有规则必须**可在 `tailwind.config.ts` 一次性配置**或**通过 className 直接表达**。

### 5.1 Type Scale（5 档够用，不要 8 档）

| 用途 | Tailwind class | Px (16 base) | 何时用 |
|---|---|---|---|
| 屏幕大标题 / hero H1 | `text-4xl font-semibold` | 36 | landing 第一行、完成屏 |
| 区块标题 / H2 | `text-2xl font-semibold` | 24 | "📝 我的文案" / "🎬 我的镜头" |
| 卡片标题 / H3 | `text-lg font-medium` | 18 | 热点卡片标题、镜头编号 |
| 正文 body | `text-base` | 16 | 所有 prose、台词、说明 |
| 辅助 caption | `text-sm text-neutral-600` | 14 | "做过 3 次"、"3 分钟前更新" |

**规则**：
- 不出现 < 14px 的字（宝妈电脑端 + 老花眼前置）
- 字重只用 `font-normal` (400) / `font-medium` (500) / `font-semibold` (600) — 不用 700 / 800
- 行高用 Tailwind 默认（不要自己调 `leading-*`）

### 5.2 Color · 暖 vs 冷 的辩护

#### 5.2.1 最终决定：**暖色为主，蓝灰仅作信息辅助**

| 角色 | 颜色 | Tailwind | 何时用 |
|---|---|---|---|
| **主品牌色（暖）** | 砖橙 / 暖陶 | `bg-orange-500` (`#f97316`) | CTA 按钮、当前选中、完成屏图标 |
| 主品牌色 hover | 深陶 | `bg-orange-600` (`#ea580c`) | 按钮 hover |
| 主品牌色 soft | 米橙 | `bg-orange-50` (`#fff7ed`) | 选中卡片背景 |
| **辅助色（冷·克制）** | 中性蓝灰 | `text-slate-700` (`#334155`) | 标题、正文 |
| 成功 | 草绿 | `text-emerald-600` (`#059669`) | "保存好了" 标签 |
| 警告 | 琥珀 | `text-amber-600` (`#d97706`) | "等你看看" 标签 |
| 失败 | 砖红 | `text-rose-600` (`#e11d48`) | "这张失败了" 标签 |
| 背景 | 暖白 | `bg-stone-50` (`#fafaf9`) | 整体页面背景 |
| 卡片白 | 纯白 | `bg-white` | 所有卡片 |
| 边框 | 暖灰 | `border-stone-200` (`#e7e5e4`) | 卡片描边、分隔线 |

**绝不使用**：
- ❌ 深蓝紫 / 渐变紫 / 霓虹蓝（即梦/可灵的"AI 科技感"，是 Cascade 要躲开的视觉信号）
- ❌ 纯黑背景（暗色科技感）
- ❌ 任何渐变（gradient）— Phase 1 全部纯色

#### 5.2.2 为什么是暖色而不是冷色

| 维度 | 冷色（蓝紫）| 暖色（橙陶） | 选 |
|---|---|---|---|
| 竞品占位 | 即梦 / 可灵 / Sora 2 全部冷色"AI 黑科技"（trend §1） | 剪映偏暖但被抖音红淹没 | **暖** — 红海vs蓝海 |
| 目标用户视觉惯性 | 工程师 / 极客 / B 端 | 宝妈每天看的小红书封面、生活博主 | **暖** |
| 品牌三关键词 | "看懂 / 接手 / 留住" 全是**关系**词 | 关系词需要**人情**，不是**科技** | **暖** |
| 反例验证 | Notion / Linear 冷色 → 工程师产品 | 小红书 / 即刻 / 大众点评 暖色 → C 端生活 | **暖** |

**一行结论**：暖色是 Cascade **拒绝在"AI 黑科技"红海里被埋没**的视觉宣言。

### 5.3 Spacing（4 档够用）

| Tailwind class | px | 何时用 |
|---|---|---|
| `gap-2 / p-2` | 8 | 卡片内紧凑元素之间（标题 ↔ 标签） |
| `gap-4 / p-4` | 16 | 卡片内常规间距、按钮内 padding |
| `gap-6 / p-6` | 24 | 卡片之间、区块内的子组件 |
| `gap-12 / py-12` | 48 | 大区块之间（hero ↔ 卡片流） |

**规则**：
- 默认垂直布局（`flex flex-col gap-6`），不要水平 hairline 拥挤
- 单行最大宽度 `max-w-2xl`（672px）— 宝妈不读长行
- 卡片圆角统一 `rounded-2xl`（16px）— 一个值，不混用 4/8/12

### 5.4 Illustration / Icon 风格

| 元素 | 来源 | 风格约束 |
|---|---|---|
| Icon | **Lucide React**（与 Radix 同生态） | 1.5px stroke、不带 fill、单色用品牌主色或 slate-700 |
| Hero 视觉 | **真实抖音/小红书封面缩略图**（5 张精选热点卡片） | 不画矢量插画、不放 AI 渲染图 — UX §3.2 "视觉要暖：人 + 厨房 + 娃，不是 AI/科技感" |
| Loading / empty state | **简单 emoji + 一句话** | 不画自定义插画（Phase 1 没设计师） |
| Logo | （Phase 1 暂用文字 wordmark） | 「Cascade」无衬线 + 主色橙；招到设计师后再做 mark |

**绝不**：
- ❌ Midjourney 风的 hero 渲染图
- ❌ 3D isometric 插画
- ❌ 任何"机器人 / 大脑 / 神经网络"的视觉隐喻

### 5.5 一行总结（可贴在工程师显示器上）

> **Tailwind + Radix · 5 字号 · 1 主色（橙）· 4 spacing 档 · `rounded-2xl` · Lucide icons · 不要渐变 · 不要插画 · 不要紫**

## 6. Loading & Failure State Copy（每个状态一句话）

> **原则**：Karpathy reviewer §2.3 "100% failure recoverability" + Jobs §9 "用户始终知道下一步"。**每个状态必须有一句中文，并且必须给下一步动作。**

### 6.1 Loading（等待）

| 场景 | 一句话 copy | 备注 |
|---|---|---|
| URL 浅分析中 | `正在看这条为什么火...` | streaming 出文字，不要空白圈圈 |
| URL 浅分析中（>15s） | `这条有点复杂，再给我 10 秒...` | 进度感升级 |
| 赛道改写中 | `正在帮你改成宝妈辅食版...` | 用她选的赛道名拼接 |
| 画镜头草稿图 | `正在画第 2 张（共 3 张）...` | 必须显示进度 X/Y |
| 画镜头草稿图（>30s/张） | `这张稍慢，再等一下...` | 不要"超时" |
| 锚点保存中 | `正在把"妈妈"存起来...` | 用具体角色名 |
| 发布包生成中 | `正在打包发布素材...` | |

**规则**：
- 不用 "Loading..." / "Processing..."
- 不用旋转圆圈作为唯一信号（必须有文字）
- 等待 > 30 秒必须升级 copy（让她知道还有人在干活）

### 6.2 Empty State（空）

| 场景 | 一句话 copy | CTA |
|---|---|---|
| 首页未登录 | `看到刷屏的视频，想做一条自己的？挑一张开始 ↓` | 3 张热点卡片 |
| 首页登录后第一次 | `今天宝妈在抖音最爱看的，是这 3 条 ↓` | 3 张热点卡片 |
| 首页登录后第 N 次（已有 run） | `继续上次的「5 点起床实录」？` | [继续] [挑新的] |
| "我的角色" 空 | `你还没存过角色 — 做一条视频，"妈妈"就会自动存下来` | （引导回首页） |
| "我的场景" 空 | `你还没存过场景 — 做一条视频，"厨房"就会自动存下来` | （引导回首页） |
| "我的视频" 空 | `还没做过 — 挑一张开始 →` | 跳首页 |

### 6.3 Success（成功 / 完成）

| 场景 | 一句话 copy | 下一步 |
|---|---|---|
| 单张镜头画好 | `画好了 — 不像你？[换张图]` | 单张操作 |
| 整条视频完成屏 | `你的「{赛道名}版{标题}」做好了 ✓` + `你做的"妈妈"和"厨房"已经存好了，下次还能用 →` | [复制标题] [复制文案] [复制标签] [下载图片] |
| 锚点首次保存 | `"妈妈本人"已经存好了 — 下次做新视频可以直接用` | 完成屏上方 toast |

### 6.4 Failure（失败 — 8 类，对齐 Karpathy reviewer §3.9）

每个失败都必须给 **3 选 1** 兜底（reviewer §3 红线 + UX §1.7）。不要只显示错误信息。

| 失败类型 | 用户看到的 copy | 3 选 1 兜底 |
|---|---|---|
| **F-1 URL 无效 / 无法解析** | `这条链接看不到 — 可能是被删了，或者不是公开视频` | [换一条] [挑精选卡片] [告诉我们] |
| **F-2 URL 不是视频** （如小红书图文笔记） | `这是张图文笔记，做不了视频参考 — 找一条视频链接试试` | [换一条] [挑精选卡片] |
| **F-3 浅分析失败 / schema 缺字段** | `这条视频的爆款公式没看清楚 — 换一条试试？` | [换一条] [挑精选卡片] [先跳过分析] |
| **F-4 改写脚本失败 / LLM 输出不合规** | `脚本写得不太顺手 — 让我再写一次？` | [再写一次] [换种风格] [我自己写] |
| **F-5 镜头草稿图失败 / Provider 超时** | `这张画失败了 — 要不要换个风格再画一次？` | [换风格再画] [跳过这张] [整段重做] |
| **F-6 镜头草稿图失败 / 内容审核** | `这张系统说不让画 — 换种描述试试？` | [换说法再画] [跳过这张] |
| **F-7 锚点保存失败** | `"妈妈"暂时没存上 — 别担心，发布素材还能用` | [再试一次] [先继续] |
| **F-8 发布包导出失败** | `发布素材没打包好 — 一张一张下载也行` | [再试一次] [手动复制] |

**绝不**：
- ❌ "Generation failed" / "Network error" / "500 Internal Server Error"
- ❌ 错误代码（"Error: PROV_TIMEOUT_402"）
- ❌ 没有下一步的报错框（"重试" 不算下一步，要给具体 3 个选项）

### 6.5 Quota / 商业化提示（Phase 1 仅暴露 2 档）

## 7. "What we are NOT" · 三条反定位

> 用于 landing page 的 "vs 别人" 解释段（如果有）、内部 onboarding、对外接受访谈时使用。**每一条都要把竞品名直接说出来**——这是 Jobs §1 + Elon §4.2 的明确指令。

### 7.1 We are NOT 即梦 / 可灵 / Sora 2

> **他们做"生成"，我们做"复刻"。**
> 他们给你一个 prompt 框让你描述视频，但你不知道"什么样的视频会火"。Cascade 反过来——你给我一条已经火了的视频，我帮你做成你的版本。

（trend §1.1 / §1.2 / §6.1）

### 7.2 We are NOT 新榜 / 飞瓜 / 卡思

> **他们给你数据，我们替你做事。**
> 他们告诉你"今天什么在火"，但看完你还要回到剪映/抖音从头开始。Cascade 把"看懂"和"做出来"合并成一步——看完一条爆款 30 分钟就出第一稿。

（trend §1.7 / vision §1.1）

### 7.3 We are NOT 剪映 / CapCut

> **他们是剪刀，我们是出门前的镜子。**
> 剪映帮你把已经拍好的素材剪成视频。Cascade 在你拍之前——帮你看懂别人为什么火、写出适合你赛道的脚本、画好关键镜头草稿。**你拍完还是回剪映剪**——我们不抢这一步。

（trend §1.4 / §3.3 — 真人出镜是信任来源，Cascade 不取代拍摄）

### 7.4 反定位使用规则

- **不主动黑同行**——用 "他们做 X / 我们做 Y" 结构，不用 "他们不行 / 我们更好"
- 三句话的句式必须一致："**他们…，我们…**"
- landing page 如果放，只放一条（不要 3 条堆一起 — 选 7.3 vs 剪映最有力，因为目标用户日常在用）
