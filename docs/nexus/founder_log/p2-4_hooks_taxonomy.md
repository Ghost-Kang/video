## 0. 为什么需要钩子分类法

**问题**: P1-3 fixture 改写发现了一个跨工单设计漏洞 — `prompts/rewrite_<niche>.md` 在真空里写,没有引用任何已验证的爆款钩子。结果 fixture 改写出来的开场全是"你家宝宝是不是也这样" 这种泛化疑问句,不命中任一 niche 的主流模板。

**解法**: 把 P2-4 已验证的爆款钩子结构化成 H1–H9 九个模式,作为强制 input 注入 LLM rewrite prompt 的 system 末尾。每个 niche 再叠加自己的 priority map,告诉 LLM 哪几个钩子是该 niche 的高 ROI 主流。

**Invariant**: LLM 改写的 shot 1 必须命中该 niche priority map 中的至少 1 个钩子,机械可校验。

---

## 1. 九个钩子模式定义

### H1 — 月龄/年龄/区间钩

**定义**: 标题或开场显式提及婴幼儿月龄、儿童年龄或年龄区间(0-12月、3-5岁、一岁加 26 斤)。

**心理机制**:
- 月龄是辅食/育儿赛道的"身份认证钩",家长会自动对号入座
- 用户在评论区高频提问"X 月可以吃吗",月龄前置直接 prefilter 受众

**触发示例**:
- @阿倩 "来看看 **12 龄宝宝** 一周辅食不重样"
- @星樾成长日记 "**六月龄宝宝**辅食添加的顺序"
- 小红书《**0-12 月**宝宝成长规律 新手妈妈必收藏》(1.8w赞)

**反面教材**:
- ❌ "宝宝爱吃的辅食" (无月龄锚点,人群泛化)
- ❌ "新手妈妈必备" (年龄缺失)

**机械化约束**:
- 正则 `^.{0,30}?(?:\d+|[零一二三四五六七八九十两])(?:个?月龄?|岁)`
- shot 1 strings 中至少匹配一次

---

### H2 — 一周不重样 / X 天清单 / 数字清单

**定义**: 标题包含数字 + 时间单位 / 数字 + 名词清单(7 道菜、6 款粥、4 个坏毛病)。

**心理机制**:
- 数字承诺 = 完整方案 = 高收藏率
- "一周不重样" 是辅食/家庭厨房赛道的 P0 模板,转化率最高

**触发示例**:
- @阿倩 "一周辅食**不重样**"
- @Ella的饭饭日记 "分享一岁宝宝**一周辅食不重样**,每天都是新口味"
- 小红书《**六款**营养又好喝的粥,**一周不重样**》(2.3w赞 / 2w收藏)
- 小红书《**7 大**危险玩具,千万别让孩子玩》(6.9w赞)

**反面教材**:
- ❌ "好喝又营养" (无数字清单,纯形容词)
- ❌ "几款粥的做法" (数字模糊化)

**机械化约束**:
- 正则 `(?:一周|7天|[0-9]+款|[0-9]+道|[0-9]+大|[0-9]+个)`
- 在标题或 shot 1 中至少匹配一次,且数字 ≥ 3 (单数不算清单)

---

### H3 — 结果承诺 / 时间承诺 / 收藏诱导

**定义**: 标题给出可量化的结果(身高蹭蹭涨 / 5 分钟搞定 / 三天瘦 4 斤)或显式收藏诱导("先收藏了"/"必收藏")。

**心理机制**:
- 结果承诺 = 用户行动动机
- 收藏诱导 = 直接抬高收藏率指标 = 算法给量

**触发示例**:
- @小糯米爸辅食记 "辅食助力,**个子蹭蹭涨**!问我拿食谱噢!"
- @王刚 "厨师长教你做一道家常菜:'家常豆腐',家常味浓郁,**先收藏了**"
- 小红书《一周不重样宝宝早餐!**5 分钟**搞定的早餐饼》
- 小红书《0-12 月宝宝成长规律,**新手妈妈必收藏**》

**反面教材**:
- ❌ "好吃又营养" (结果模糊)
- ❌ "随便分享一下" (反诱导,降权)

**机械化约束**:
- 关键词列表: `["蹭蹭涨", "瘦", "搞定", "学会", "必收藏", "先收藏", "建议收藏", "拿食谱", "教你", "告诉你"]`
- 标题 + shot 1 + shot N (最后) 中至少匹配一次

---

### H4 — 反常识 / 反差 / 危机制造 / 稀缺感

**定义**: 标题制造认知冲突,可分四个子类:
- **反常识**: "宽油是什么油"(违反直觉)
- **反差**: "餐厅 88 我做 12"(数字反差)
- **危机制造**: "千万别让孩子玩"(警示)
- **稀缺感**: "复刻故宫餐厅饮品"(权威 + 难得)

**心理机制**:
- 制造认知缝隙 → 用户必须看完才能闭合 → 完播率拉满
- 是家庭厨房 / 育儿安全赛道的 P0 模板

**触发示例**:
- @王刚 "家常豆腐"(评论区"宽油是什么油"成 7 年长尾梗)
- 小红书《**复刻故宫**餐厅饮品-馥郁杨梅妃子饮》(25w赞)
- 小红书《**7 大危险玩具**!!**千万别**让孩子玩》(6.9w赞,危机制造)
- 小红书《这是一场大米饭的**华丽变身**》(6.6w赞,反差悬念)

**反面教材**:
- ❌ "家常菜怎么做" (无认知冲突)
- ❌ "玩具推荐" (无危机,无稀缺)

**机械化约束**:
- 关键词列表: `["千万别", "绝对不能", "复刻", "竟然", "原来", "没想到", "为什么", "VS", "vs", "对比"]` + 数字差异检测 (两个数字之间差距 ≥ 5x)
- 标题 + shot 1 中至少匹配一次

---

### H5 — IP 系列化 / 差异化人称

**定义**: 用固定人物 IP / 系列化标题模板 / 差异化人称视角(爸爸视角 / 丈夫视角 / 师傅视角)。

**心理机制**:
- IP 化 = 二次回流(粉丝认人不认内容)
- 差异化人称 = 在饱和赛道里突出(辅食 niche 99% 是宝妈视角,爸爸视角天然差异化)

**触发示例**:
- @老饭骨 "看**马爷**做河间包肉"(IP 系列"X 爷做菜")
- @小糯米爸辅食记 (爸爸视角辅食号)
- @钻爸的古村落生活 (爸爸视角育儿号)

**反面教材**:
- ❌ 完全无人称 ("一道好吃的菜")
- ❌ 替换性人称 ("我家做这道菜",无 IP 锚点)

**机械化约束**:
- shot 1 或博主签名中需含: 固定称呼(爷/叔/师傅/X 爸 X 妈) 或系列前缀(VlogX、第 X 集)

---

### H6 — 节日 / 系列化场景 / 萌娃 IP

**定义**: 用节日(过年篇/开学篇/六一篇)或系列场景(放学后/吃晚饭/睡前)或萌娃命名(妍妍肉肉/小糯米)创造可持续选题。

**心理机制**:
- 节日和场景天然可重复 → 系列内容矩阵
- 萌娃 IP = 育儿赛道的免费流量增益

**触发示例**:
- @元旦快乐的妈 "新手妈妈带娃日常,**回家过年篇**"
- @妍爸妍妈 "记录**妍妍肉肉放学后**的 vlog"

**机械化约束**:
- 节日关键词: `["过年", "中秋", "端午", "六一", "新年", "开学"]` 或
- 场景关键词: `["放学后", "睡前", "晨间", "晚餐后"]` 或
- 命名锚点: 标题中含独立人名/昵称 ≥ 1 个 (非"宝宝/孩子/娃"通用词)

---

### H7 — 家庭温情 / 情感流 / 烟火气

**定义**: 标题或开场用日常温情、家庭场景、烟火气营造情绪共鸣(不是宣泄式,是温暖式)。

**心理机制**:
- 温情流是育儿/家庭厨房中年用户的 P0 流量来源
- 评论区高粘性 → 算法给量

**触发示例**:
- @钻爸 "当孩子读的书,比我烧过的菜还多"
- @沈家三叔 "三叔盘出家常菜红烧肉,**再忙也要好好吃饭**"
- @华华 "老公生日,华华晚上准备海鲜大餐,**一家人围着庆祝**"

**与 H8 的区别**:
- H7 是温暖式,基调是"家很美好"
- H8 是宣泄式,基调是"妈妈很累/很委屈"

**机械化约束**:
- 关键词列表: `["一家人", "好好吃饭", "再忙也", "家的味道", "妈妈的", "温馨", "烟火气", "再累也"]`
- 至少匹配一次,且整体情绪不带"崩溃/累/苦"等负面关键词

---

### H8 — 情绪共鸣 / 委屈宣泄 / 痛点叙事

**定义**: 标题或开场直接陈述妈妈的疲惫 / 委屈 / 崩溃,触发同温层共鸣。

**心理机制**:
- 不是问解决方案,是问"你也这样吗"
- 评论区高互动密度,共鸣留言("我也是")为主

**触发示例**:
- @当妈以后 "**当妈以后才发现**,最累的不是熬夜带娃,**而是没人理解你的委屈**"
- @素喵喵 "永远不会有人知道**一个人带娃的心酸**"

**机械化约束**:
- 关键词列表: `["当妈以后", "才发现", "没人懂", "崩溃", "委屈", "心酸", "都懂", "都明白"]`
- 至少匹配一次

---

### H9 — 评论区二次梗钩 [新增,来自 P1-3 F-3-b]

**定义**: 在内容中**故意埋一个违反直觉的小技术点 / 反常识细节**,作为评论区话题种子,让粉丝在评论区自发玩梗。

**心理机制**:
- 评论区每次有人提到这个梗 → 算法判定为高互动 → 给量
- 是家庭厨房 / 育儿科普赛道的"长尾流量发动机"
- 评论区玩梗 → 视频 7 年后还在涨赞 (王刚家常豆腐就是典型案例)

**触发示例**:
- @王刚 "宽油是什么油" → 评论区 7 年都在问/玩
- 假设性: "为什么牛肉要逆纹切" / "为什么蒸蛋要过筛两次" / "为什么辅食米粉要冲到能挂勺"

**与其他钩子的区别**:
- H4 反常识是**开场制造冲突**,H9 是**内容中埋种子**
- H4 触发的是完播,H9 触发的是评论区长尾

**机械化约束**:
- shot 2~N (中段) 必须有 ≥1 个"反常识技术点"
- 检测: 句子模式 `(?:为什么|不是|偏要|其实).{2,15}?(?:不|要|得|必须)`
- 或 `(?:很多人|大家|其实).{2,15}(?:都错了|搞反了|不知道)`

## 2. 每 niche 的 Priority Map

LLM rewrite prompt 必须按 priority map 选择钩子。Priority 越高,在 shot 1 必须使用的概率越高。

| **P0 必选** | H1 (月龄) | shot 1 must contain |

## 3. LLM Prompt 注入模板

每个 `prompts/rewrite_<niche>.md` 在 system prompt 末尾加入:

```markdown
## 钩子模式约束 (must follow)

你正在改写 {niche} 赛道的内容。本 niche 的 priority map 如下:

{paste priority map for this niche}

### 改写规则

1. shot 1 必须命中 priority map 中 P0 必选钩子的至少一个 (机械可校验)
2. 全脚本必须命中 priority map 中 ≥2 个钩子,其中至少 1 个 P0
3. 不得使用 priority map 中标记为"反面"的钩子模式
4. 钩子定义详见 `docs/nexus/founder_log/p2-4_hooks_taxonomy.md`,改写前必须读取

### 自检 checklist (LLM 输出前必填)

输出 JSON 时附带:

```json
{
  "hooks_used": ["H1", "H2"],
  "hook_per_shot": {
    "shot_1": ["H1", "H2"],
    "shot_2": [],
    "shot_3": ["H3"],
    "shot_4": ["H7"]
  },
  "priority_compliance": "P0 hooks H1+H2 both hit in shot 1 ✅"
}
```

如果自检不过,直接重新生成,不要输出不合规的脚本。
```

## 4. 机械检查项 (P2-6 eval harness)

注入 `backend/tests/test_rewrite.py`,在原 5 项机械检查上新增:

| ID | 检查项 | 通过条件 |
|---|---|---|
| #6 | `visual_diversity_score` | 同脚本内 shot 间视觉描述 token 重合率 ≤ 50% (来自 P1-3 §4-A) |
| #7 | `nutrient_category_consistency` | 仅 baomam_fushi: 替代食材必须与原食材同营养类目 (来自 F-1-b) |
| #8 | `hook_p0_compliance` | shot 1 必须命中该 niche P0 钩子至少一个 (来自本文档 §2) |
| #9 | `hook_diversity` | 全脚本至少命中 2 个不同 H 钩子 |
| #10 | `negative_hook_absence` | 不得出现该 niche priority map 中的"反面"钩子 |

**通过门槛**: 10 项中至少 8 项通过,且 #7/#8 是 P0 必过项。
