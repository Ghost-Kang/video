**⚠️ Lessons learned (写进 P2-4 SOP)**:
1. 搜索引擎 snippet 中的赞数 100% 不可信(被搜索页相邻视频的数据错配)
2. 抖音对未登录爬虫隐藏赞数,真实信号只有评论区
3. 验证爆款的唯一可靠信号: **评论数 ≥10 + 跨时间分布 + 评论内容多样性**
4. 头部 IP (王刚/老饭骨等) 的早期视频是最稳定的真爆款样本源

## PM 用法 (W2D2)

1. PM 跑 `CASCADE_REWRITE_UPSTREAM=llm` 处理这 15 条
2. **LLM prompt 必须显式枚举 H1-H8 钩子模式**,不要让模型自己抽象
3. 改写时**保留原视频的钩子模式 ID** 作为 metadata 落盘
4. 输出落到 `docs/nexus/founder_log/p2-4_llm_outputs_<date>/`,每个 niche 一个子目录
5. **育儿 #1 (妮妮) 单独处理**: LLM 改写时输入"NEGATIVE PROMPT",学习"不要这样写"
6. **厨房 #4 (二伯快乐油) 单独处理**: 作为"产品融入内容"的边界样本
7. Founder 在 `docs/nexus/founder_log/p2-4_qualitative_signoff_<date>.md` 第二轮签字
8. 若 4/5 通过 founder qualitative bar → P2-4 done → 触发 P2-6 eval harness

## 复盘 (founder 写给未来的自己)

**信号源演化路径**:
1. ❌ 通用关键词搜索 (命中率 13%) — snippet 数据陷阱
2. ⚠️ 评论密度补救 (命中率 40%) — 局部修复
3. ✅ 头部 IP 名 + 评论密度 (命中率 85%) — 黄金路径

**写进 P2-X 通用 SOP**:
- 找抖音爆款样本 → 优先搜已知头部 IP 名 (王刚/老饭骨/年糕妈妈/妮妮等)
- 验证爆款 → 一律用评论密度 + 跨时间分布,绝不信 snippet 赞数
- 多样性兜底 → 主流 H1-H7 + 至少 1 条反面教材(❌-IP) + 1 条边界样本(EDGE CASE)
