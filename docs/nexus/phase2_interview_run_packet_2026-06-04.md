## 1. 招募 DM(复制即发,目标现有邀请码 cohort + 真实抖音/短视频创作者)

> 招 12 人(命中 ≥10 试用 / ≥5 完成闭环 / ≥3 一周内回访)。不公网招(避免污染漏斗 + 成本归因)。

```
嗨,在做一个帮短视频创作者「把刷到的爆款 1 分钟拆透 + 改成你自己脚本」的小工具,内测中。
想请你花 45 分钟,边用边聊:你平时怎么找选题/拆爆款/改写,卡在哪。
不是让你学新工具,就想看你真实的工作流 + 哪一步最费劲。
有个小酬谢 🧧。这周方便吗?(线上腾讯会议/线下都行)
```

## 2. 1 页 runsheet(45–60 min,半结构化 + 情境法)

| 阶段 | 时长 | 主攻 | 关键动作 / 问 |
|---|---|---|---|
| 0 暖场+同意 | 5 | — | 说明会录音;确认知情同意 |
| 1 现工作流 | 10 | H1 | 「你最近发的一条,从想法到发出去怎么做的?**哪一步最痛?**」(逐字记痛点) |
| 2 看到爆款第一步 | 10 | H1→H2 | 甩一条爆款,**现场观察**:他第一反应做什么?(不引导) |
| 3 「为什么火」有没有用 | 10 | H2 | 让他用工具拆这条 → 「这个拆解,哪句对你有用/没用?」 |
| 4 改写到什么程度愿用 | 10 | H3 | 让他点「改成我的版本」→ **「这版你会直接发吗?差在哪?」**(改写质量金标准) |
| 5 愿不愿发 AI 生成镜头图 | 8 | H5 | 看草稿图 → 「这种图你会拿去发吗?」 |
| 6 端到端+收尾 | 7 | — | 「整条用下来,愿意一周用几次?**愿意付 ¥39/月吗?**」 |
| 7 失败恢复(机会触发) | — | H7 | 如果中途真失败了,看他**下一步**做什么 |

**北极星(每场必答)**:用他自己的话填 `value_statement_match` —— 他会怎么一句话跟朋友介绍这工具?

## 3. 每场落点(收口 = 一条命令落库)

访谈后,把 4 个必填字段填进下面命令(`value_statement_match` / `would_pay_39` / `notes_url` / `niche`),跑一条即落 events 表:

```bash
# 注意 X-Invite-Code 头(/api/events 在邀请码门后,缺它 401)。code = 当前邀请码 cascade。
curl -s -X POST http://119.29.18.216/api/events \
  -H "Content-Type: application/json" \
  -H "X-Invite-Code: cascade" \
  -d '{
    "event_name": "interview_logged",
    "user_id": "intv-<受访者代号>",
    "run_id": "intv-2026-06-xx",
    "payload": {
      "value_statement_match": "<他自己的一句话价值描述>",
      "would_pay_39": true,
      "notes_url": "<腾讯文档/飞书 访谈笔记链接>",
      "niche": "<美食/健身/知识/…>"
    }
  }'
```

- `notes_url` 指向结构化笔记(按指南 §4.2:现工作流痛点逐字 / 改写弃稿点 / 「足够好的改写」用户原话)。
- 回访(第 7±2 天)再 emit 一条(`run_id` 带 `-followup`),喂「一周内回访 ≥3」。
